## Problem Statement

You are given an array of `k` linked-lists `lists`, each linked-list is sorted in ascending order.

_Merge all the linked-lists into one sorted linked-list and return it._

 

**Example 1:**

```
Input: lists = [[1,4,5],[1,3,4],[2,6]]
Output: [1,1,2,3,4,4,5,6]
Explanation: The linked-lists are:
[
 1->4->5,
 1->3->4,
 2->6
]
merging them into one sorted linked list:
1->1->2->3->4->4->5->6
```

**Example 2:**

```
Input: lists = []
Output: []
```

**Example 3:**

```
Input: lists = [[]]
Output: []
```

 

**Constraints:**

- `k == lists.length`
- `0 <= k <= 10^4`
- `0 <= lists[i].length <= 500`
- `-10^4 <= lists[i][j] <= 10^4`
- `lists[i]` is sorted in **ascending order**.
- The sum of `lists[i].length` will not exceed `10^4`.

## Solution

```python
from typing import List, Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        return self.merge(lists, 0, len(lists) - 1)

    def merge(self, lists, start, end):
        if start == end:
            return lists[start]
        mid = start + (end - start) // 2
        left = self.merge(lists, start, mid)
        right = self.merge(lists, mid + 1, end)
        return self.merge_two_lists(left, right)

    def merge_two_lists(self, left, right):
        dummy = ListNode()
        current = dummy
        while left and right:
            if left.val < right.val:
                current.next = left
                left = left.next
            else:
                current.next = right
                right = right.next
            current = current.next
        current.next = left or right
        return dummy.next
```


## Explanation

### Approach: Divide-and-Conquer Pairwise Merging

This implementation recursively splits the list of linked lists into two halves,
merges each half, then merges the two sorted results.

It is analogous to merge sort on lists.

### Why This Is Efficient

If total nodes are `N` across `k` lists:
- each merge level processes all `N` nodes once
- number of levels is `log k`

So total time is `O(N log k)`.

### Code Structure

```python
left = self.merge(lists, start, mid)
right = self.merge(lists, mid + 1, end)
return self.merge_two_lists(left, right)
```

`merge_two_lists` is standard linear two-pointer merge using a dummy head.

### Two-List Merge Core

```python
while left and right:
    if left.val < right.val:
        current.next = left
        left = left.next
    else:
        current.next = right
        right = right.next
    current = current.next
current.next = left or right
```

### Why This Works

- Base case (`start == end`) returns one already-sorted list
- Recursive combining preserves sorted order at every level
- Eventually all lists are merged into one globally sorted list


## Complexity

- **Time**: `O(N log k)`
- **Space**: `O(k)`
