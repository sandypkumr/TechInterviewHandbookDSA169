## Problem Statement

Given `n` non-negative integers representing an elevation map where the width of each bar is `1`, compute how much water it can trap after raining.

 

**Example 1:**

```
Input: height = [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6
Explanation: The above elevation map (black section) is represented by array [0,1,0,2,1,0,1,3,2,1,2,1]. In this case, 6 units of rain water (blue section) are being trapped.
```

**Example 2:**

```
Input: height = [4,2,0,3,2,5]
Output: 9
```

 

**Constraints:**

- `n == height.length`
- `1 <= n <= 2 * 10^4`
- `0 <= height[i] <= 10^5`

## Solution

```python
from typing import List


class Solution:
    def trap(self, height: List[int]) -> int:
        stack = []
        water = 0
        for i, h in enumerate(height):
            while stack and h > height[stack[-1]]:
                top = stack.pop()
                if not stack:
                    break
                distance = i - stack[-1] - 1
                water += distance * (min(h, height[stack[-1]]) - height[top])
            stack.append(i)
        return water
```


## Explanation

### Approach: Monotonic Stack

This solution uses a **monotonic decreasing stack** to identify bounded valleys where water can accumulate.

### Intuition

Water above a bar is trapped only when there is a taller bar on both the left and the right. When a new bar is taller than the one on top of the stack, it may close a container, and the popped bar becomes the basin floor whose trapped water can now be computed.

### Algorithm

Here's the step-by-step approach:

1. Traverse the height array while storing indices of bars in decreasing-height order.
2. When the current bar is taller than the stack top, pop the top as a potential basin floor.
3. If a left boundary still exists, compute the trapped water using the distance between boundaries and the bounded height.
4. Continue until the stack is valid again, then push the current index.

### Code Breakdown

The variable `top` represents the bottom of a trapped region. After popping it, the current index is the right boundary and the new stack top is the left boundary, which is exactly the information needed to compute the trapped water above `top`.

### Why This Works

- **Valid basin detection:** A trapped region exists only when both left and right boundaries are present.
- **Exact area formula:** `distance * bounded_height` measures the water above the popped basin floor.
- **Linear processing:** Each bar is pushed and popped at most once, so all trapped regions are accounted for efficiently.


## Complexity

- **Time**: `O(n)`
- **Space**: `O(1)`
