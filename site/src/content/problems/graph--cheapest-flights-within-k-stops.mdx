## Problem Statement

There are `n` cities connected by some number of flights. You are given an array `flights` where `flights[i] = [from_i, to_i, price_i]` indicates that there is a flight from city `from_i` to city `to_i` with cost `price_i`.

You are also given three integers `src`, `dst`, and `k`, return _**the cheapest price** from _`src`_ to _`dst`_ with at most _`k`_ stops. _If there is no such route, return_ _`-1`.

 

**Example 1:**

```
Input: n = 4, flights = [[0,1,100],[1,2,100],[2,0,100],[1,3,600],[2,3,200]], src = 0, dst = 3, k = 1
Output: 700
Explanation:
The graph is shown above.
The optimal path with at most 1 stop from city 0 to 3 is marked in red and has cost 100 + 600 = 700.
Note that the path through cities [0,1,2,3] is cheaper but is invalid because it uses 2 stops.
```

**Example 2:**

```
Input: n = 3, flights = [[0,1,100],[1,2,100],[0,2,500]], src = 0, dst = 2, k = 1
Output: 200
Explanation:
The graph is shown above.
The optimal path with at most 1 stop from city 0 to 2 is marked in red and has cost 100 + 100 = 200.
```

**Example 3:**

```
Input: n = 3, flights = [[0,1,100],[1,2,100],[0,2,500]], src = 0, dst = 2, k = 0
Output: 500
Explanation:
The graph is shown above.
The optimal path with no stops from city 0 to 2 is marked in red and has cost 500.
```

 

**Constraints:**

- `2 <= n <= 100`
- `0 <= flights.length <= (n * (n - 1) / 2)`
- `flights[i].length == 3`
- `0 <= from_i, to_i < n`
- `from_i != to_i`
- `1 <= price_i <= 10^4`
- There will not be any multiple flights between two cities.
- `0 <= src, dst, k < n`
- `src != dst`

## Solution

```python
from collections import defaultdict, deque
from typing import List


class Solution:
    def findCheapestPrice(self, n: int, flights: List[List[int]], src: int, dst: int, k: int) -> int:
        graph = defaultdict(list)
        for u, v, w in flights:
            graph[u].append((v, w))
        dist = [float('inf')] * n
        dist[src] = 0
        queue = deque()
        queue.append((src, 0))
        stops = 0
        while queue:
            for _ in range(len(queue)):
                city, cost = queue.popleft()
                for neighbor, price in graph[city]:
                    if cost + price < dist[neighbor]:
                        dist[neighbor] = cost + price
                        queue.append((neighbor, cost + price))
            stops += 1
            if stops > k:
                break
        return dist[dst] if dist[dst] != float('inf') else -1
```


## Explanation

### Approach: BFS

This solution uses **Breadth-First Search (BFS)** with a queue to explore states level by level.

### Intuition

The stop limit matters more than raw edge count, so the search expands the graph level by level, where each level represents taking one more flight. At the same time, `dist` stores the cheapest cost seen so far for each city, preventing obviously worse paths from being expanded.

### Algorithm

Here's the step-by-step approach:

1. Build an adjacency list of outgoing flights.
2. Initialize `dist[src] = 0` and begin BFS-style traversal from `(src, 0)`.
3. For each level, relax every outgoing edge from the current cities. If a cheaper cost to a neighbor is found, update `dist[neighbor]` and enqueue it for the next level.
4. Stop after exploring at most `k + 1` flight layers and return the best cost for `dst`, or `-1` if unreachable.

### Code Breakdown

The queue stores `(city, cost)` states for the current number of flights used. The `for _ in range(len(queue))` loop freezes one level at a time, so all relaxations in that pass correspond to the same stop count before `stops` is incremented.

### Why This Works

- **Stop-bounded exploration:** Each BFS layer represents one additional flight, so the algorithm respects the maximum stop constraint.
- **Cost relaxation:** `dist` keeps only improving costs, which prunes inferior paths.
- **Complete coverage within limit:** Any valid route with at most `k` stops uses at most `k + 1` edges, and all such edge counts are explored.
- **O(1) lookups:** Hash maps provide constant-time insertion and retrieval, critical for efficient searching
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
