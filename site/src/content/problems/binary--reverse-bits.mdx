## Problem Statement

Reverse bits of a given 32 bits signed integer.

 

**Example 1:**

**Input:** n = 43261596

**Output:** 964176192

**Explanation:**

	
		
			Integer
			Binary
		
		
			43261596
			00000010100101000001111010011100
		
		
			964176192
			00111001011110000010100101000000
		
	

**Example 2:**

**Input:** n = 2147483644

**Output:** 1073741822

**Explanation:**

	
		
			Integer
			Binary
		
		
			2147483644
			01111111111111111111111111111100
		
		
			1073741822
			00111111111111111111111111111110
		
	

 

**Constraints:**

- `0 <= n <= 2^31 - 2`
- `n` is even.

 

**Follow up:** If this function is called many times, how would you optimize it?

## Solution

```python
class Solution:
    def reverseBits(self, n: int) -> int:
        result = 0
        for _ in range(32):
            result <<= 1
            result |= n & 1
            n >>= 1
        return result
```


## Explanation

### Approach: Build Reversed Number Bit-by-Bit

Read bits from `n` from right to left (least significant bit first), and append them to `result` from left to right by shifting `result` left each step.

Because this is a 32-bit problem, we do exactly 32 iterations.

### Core Operation Per Iteration

1. `result <<= 1` -> make room for next bit
2. `result |= (n & 1)` -> copy current last bit from `n`
3. `n >>= 1` -> move to next bit in `n`

### Small Example (conceptual)

Suppose bits are `...abc` (where `c` is current LSB):
- take `c` from `n`
- append to right side of `result`
- repeat for all 32 bits

After 32 steps, bit positions are fully reversed.

### Code Breakdown

```python
result = 0
for _ in range(32):
	result <<= 1
	result |= n & 1
	n >>= 1
return result
```

### Why This Works

- Each loop transfers one bit from input to mirrored position in output
- Exactly 32 iterations ensure full 32-bit reversal
- Bitwise ops are constant-time

- **Time**: `O(1)`
- **Space**: `O(1)`
