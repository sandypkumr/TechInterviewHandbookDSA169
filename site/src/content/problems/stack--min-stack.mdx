## Problem Statement

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

Implement the `MinStack` class:

- `MinStack()` initializes the stack object.
- `void push(int val)` pushes the element `val` onto the stack.
- `void pop()` removes the element on the top of the stack.
- `int top()` gets the top element of the stack.
- `int getMin()` retrieves the minimum element in the stack.

You must implement a solution with `O(1)` time complexity for each function.

 

**Example 1:**

```
Input
["MinStack","push","push","push","getMin","pop","top","getMin"]
[[],[-2],[0],[-3],[],[],[],[]]

Output
[null,null,null,null,-3,null,0,-2]

Explanation
MinStack minStack = new MinStack();
minStack.push(-2);
minStack.push(0);
minStack.push(-3);
minStack.getMin(); // return -3
minStack.pop();
minStack.top(); // return 0
minStack.getMin(); // return -2
```

 

**Constraints:**

- `-2^31 <= val <= 2^31 - 1`
- Methods `pop`, `top` and `getMin` operations will always be called on **non-empty** stacks.
- At most `3 * 10^4` calls will be made to `push`, `pop`, `top`, and `getMin`.

## Solution

```python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, val: int) -> None:
        self.stack.append(val)
        if not self.min_stack or val <= self.min_stack[-1]:
            self.min_stack.append(val)

    def pop(self) -> None:
        if self.stack.pop() == self.min_stack[-1]:
            self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]
```


## Explanation

### Approach: Main Stack + Auxiliary Min Stack

Maintain two stacks:
- `stack`: all values
- `min_stack`: monotonic stack of current minimum values

Push rule:
- push to `stack`
- push to `min_stack` only if the new value is `<=` the current minimum

Pop rule:
- pop from `stack`
- if popped value equals top of `min_stack`, pop `min_stack` too

Then `getMin()` is always `min_stack[-1]` in O(1).

### Code Core

```python
def push(self, val):
    self.stack.append(val)
    if not self.min_stack or val <= self.min_stack[-1]:
        self.min_stack.append(val)

def pop(self):
    if self.stack.pop() == self.min_stack[-1]:
        self.min_stack.pop()
```

### Why Duplicates Need `<=`

If minimum value appears multiple times, each occurrence must be tracked. Using `<=` ensures min remains correct after popping one duplicate.

### Why This Works

- `min_stack` always mirrors minima history of `stack`
- `top`, `pop`, `push`, and `getMin` all remain O(1)
