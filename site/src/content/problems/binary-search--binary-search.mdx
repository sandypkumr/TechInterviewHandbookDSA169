## Problem Statement

Given an array of integers `nums` which is sorted in ascending order, and an integer `target`, write a function to search `target` in `nums`. If `target` exists, then return its index. Otherwise, return `-1`.

You must write an algorithm with `O(log n)` runtime complexity.

 

**Example 1:**

```
Input: nums = [-1,0,3,5,9,12], target = 9
Output: 4
Explanation: 9 exists in nums and its index is 4
```

**Example 2:**

```
Input: nums = [-1,0,3,5,9,12], target = 2
Output: -1
Explanation: 2 does not exist in nums so return -1
```

 

**Constraints:**

- `1 <= nums.length <= 10^4`
- `-10^4 < nums[i], target < 10^4`
- All the integers in `nums` are **unique**.
- `nums` is sorted in ascending order.

## Solution

```python
from typing import List


class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                return mid
            if nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
        return -1
```


## Explanation

### Approach: Binary Search - Halve the Search Space

Binary search repeatedly divides the search space in half by comparing the middle element with the target. If the middle is too small, search the right half; if too large, search the left half.

### Why Binary Search Achieves O(log n)

Each comparison eliminates half the remaining candidates. After k comparisons, only n/2^k elements remain. When this reaches 1, we've found our target: n/2^k = 1 → k = log₂(n).

### Step-by-Step Walkthrough

**Input:** `nums = [-1,0,3,5,9,12], target = 9`

```
Initial: left=0, right=5, search space = [6 elements]

Iteration 1:
  mid = (0 + 5) // 2 = 2 → nums[2] = 3
  3 < 9, so search right half
  left = 3

Iteration 2:
  mid = (3 + 5) // 2 = 4 → nums[4] = 9
  9 == 9 ✓ FOUND!
  Return index 4

Total comparisons: 2 (vs 6 with linear search)
```

For array of size 1 million, binary search needs ~20 comparisons vs 1 million with linear!

### Code Breakdown

```python
mid = left + (right - left) // 2  # Avoid overflow
if nums[mid] == target:
    return mid  # Found!
elif nums[mid] < target:
    left = mid + 1  # Search right half
else:
    right = mid - 1  # Search left half
```

**Key detail:** `left + (right - left) // 2` prevents integer overflow (vs `(left + right) // 2`)

### Why Sorted Input is Essential

Binary search ONLY works on sorted data. If unsorted, comparisons don't reliably narrow the search space. This is why sort-then-binary-search approaches are common.
