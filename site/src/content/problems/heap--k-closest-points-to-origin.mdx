## Problem Statement

Given an array of `points` where `points[i] = [x_i, y_i]` represents a point on the **X-Y** plane and an integer `k`, return the `k` closest points to the origin `(0, 0)`.

The distance between two points on the **X-Y** plane is the Euclidean distance (i.e., `&radic;(x_1 - x_2)^2 + (y_1 - y_2)^2`).

You may return the answer in **any order**. The answer is **guaranteed** to be **unique** (except for the order that it is in).

 

**Example 1:**

```
Input: points = [[1,3],[-2,2]], k = 1
Output: [[-2,2]]
Explanation:
The distance between (1, 3) and the origin is sqrt(10).
The distance between (-2, 2) and the origin is sqrt(8).
Since sqrt(8) < sqrt(10), (-2, 2) is closer to the origin.
We only want the closest k = 1 points from the origin, so the answer is just [[-2,2]].
```

**Example 2:**

```
Input: points = [[3,3],[5,-1],[-2,4]], k = 2
Output: [[3,3],[-2,4]]
Explanation: The answer [[-2,4],[3,3]] would also be accepted.
```

 

**Constraints:**

- `1 <= k <= points.length <= 10^4`
- `-10^4 <= x_i, y_i <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def kClosest(self, points: List[List[int]], k: int) -> List[List[int]]:
        def distance(point):
            return point[0] ** 2 + point[1] ** 2
        return sorted(points, key=distance)[:k]
```


## Explanation

### Approach: Sorting by Distance

This solution computes each point's squared distance from the origin, sorts the points by that value, and returns the first `k`.

### Intuition

For comparing distances, we do not need square roots because `x^2 + y^2` preserves the same ordering. Once the points are sorted by squared distance, the first `k` points are exactly the `k` closest to the origin.

### Algorithm

Here's the step-by-step approach:

1. Define a helper that returns a point's squared distance from the origin.
2. Sort the list of points using that helper as the key.
3. Take the first `k` points from the sorted order.
4. Return that slice.

### Code Breakdown

The `distance` helper avoids repeated inline arithmetic and keeps the `sorted(..., key=distance)` call readable. Because only relative order matters, using squared distance is both correct and cheaper than computing Euclidean distance.

### Why This Works

- **Equivalent ordering:** Sorting by `x^2 + y^2` gives the same ranking as sorting by actual distance.
- **Sorted prefix answer:** After sorting, every point before index `k` is no farther than any point after it.
- **Simple correctness:** Returning the first `k` points therefore yields a valid set of `k` closest points.


## Complexity

- **Time**: `O(n log k)`
- **Space**: `O(k)`
