## Problem Statement

Given a string `s`, return _the longest_ _palindromic_ _substring_ in `s`.

 

**Example 1:**

```
Input: s = "babad"
Output: "bab"
Explanation: "aba" is also a valid answer.
```

**Example 2:**

```
Input: s = "cbbd"
Output: "bb"
```

 

**Constraints:**

- `1 <= s.length <= 1000`
- `s` consist of only digits and English letters.

## Solution

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n = len(s)
        if n == 0:
            return ""
        start = 0
        end = 0
        for i in range(n):
            len1 = self.expandAroundCenter(s, i, i)
            len2 = self.expandAroundCenter(s, i, i + 1)
            length = max(len1, len2)
            if length > end - start:
                start = i - (length - 1) // 2
                end = i + length // 2
        return s[start:end + 1]

    def expandAroundCenter(self, s: str, i1: int, i2: int) -> int:
        while i1 >= 0 and i2 < len(s) and s[i1] == s[i2]:
            i1 -= 1
            i2 += 1
        return i2 - i1 - 1
```


## Explanation

### Approach: Expand Around Center

This solution treats each index (and each gap between indices) as a possible palindrome center and expands outward while the characters match.

### Intuition

Every palindrome is centered either on one character (odd length) or between two characters (even length). If we expand from every possible center and keep the longest span we find, we are guaranteed to discover the longest palindromic substring.

### Algorithm

Here's the step-by-step approach:

1. Return an empty string immediately if the input string is empty.
2. For each index `i`, expand around center `(i, i)` for odd-length palindromes and `(i, i + 1)` for even-length palindromes.
3. Keep the longer of those two palindrome lengths and update the best `[start, end]` window when needed.
4. Return the substring defined by the final best window.

### Code Breakdown

The helper `expandAroundCenter` keeps moving outward as long as the mirrored characters stay equal, then returns the palindrome length it discovered. The main loop converts that length back into start and end indices for slicing the answer.

### Why This Works

- **Complete center coverage:** Every palindrome has a valid odd or even center that the algorithm checks.
- **Maximal local expansion:** Expanding until mismatch ensures each checked center contributes its longest possible palindrome.
- **Global optimum tracking:** Recording the longest palindrome seen across all centers yields the final answer.
- **Optimization:** Balances time complexity gains with reasonable space usage

- **Time**: `O(n²)`
- **Space**: `O(1)`
