## Problem Statement

Given an integer array `nums` of length `n` and an integer `target`, find three integers at **distinct indices** in `nums` such that the sum is closest to `target`.

Return _the sum of the three integers_.

You may assume that each input would have exactly one solution.

 

**Example 1:**

```
Input: nums = [-1,2,1,-4], target = 1
Output: 2
Explanation: The sum that is closest to the target is 2. (-1 + 2 + 1 = 2).
```

**Example 2:**

```
Input: nums = [0,0,0], target = 1
Output: 0
Explanation: The sum that is closest to the target is 0. (0 + 0 + 0 = 0).
```

 

**Constraints:**

- `3 <= nums.length <= 500`
- `-1000 <= nums[i] <= 1000`
- `-10^4 <= target <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def threeSumClosest(self, nums: List[int], target: int) -> int:
        nums.sort()
        closest_sum = float('inf')
        for i in range(len(nums) - 2):
            left, right = i + 1, len(nums) - 1
            while left < right:
                current_sum = nums[i] + nums[left] + nums[right]
                if abs(target - current_sum) < abs(target - closest_sum):
                    closest_sum = current_sum
                if current_sum < target:
                    left += 1
                else:
                    right -= 1
        return closest_sum
```


## Explanation

### Approach: Two Pointers

This solution sorts the input first, then uses the **Two Pointers** technique to converge efficiently.

### Intuition

After sorting, fixing one number turns the problem into finding two other numbers whose sum is as close as possible to `target - nums[i]`. Because the array is ordered, moving the left pointer increases the sum and moving the right pointer decreases it, which lets us improve the candidate efficiently.

### Algorithm

Here's the step-by-step approach:

1. Sort the array so pointer movements have predictable effects on the sum.
2. For each index `i`, place `left` at `i + 1` and `right` at the end of the array.
3. Compute the current triplet sum, update `closest_sum` if this triplet is nearer to `target`, and then move the pointer that brings the sum closer.
4. Continue until all anchors have been checked, then return the best triplet sum found.

### Code Breakdown

The outer loop chooses the first value of the triplet. The inner while loop explores all useful pairs for that fixed value without restarting from scratch, because the sorted order tells us exactly how a pointer move changes the total

### Why This Works

- **Sorted monotonicity:** Increasing `left` makes the sum larger, while decreasing `right` makes it smaller.
- **Complete search per anchor:** For each fixed `i`, the two-pointer sweep considers every relevant pair in linear time.
- **Best-so-far tracking:** Updating `closest_sum` whenever a better candidate appears guarantees the final answer is the closest triplet encountered.
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
