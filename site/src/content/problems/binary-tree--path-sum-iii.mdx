## Problem Statement

Given the `root` of a binary tree and an integer `targetSum`, return _the number of paths where the sum of the values along the path equals_ `targetSum`.

The path does not need to start or end at the root or a leaf, but it must go downwards (i.e., traveling only from parent nodes to child nodes).

 

**Example 1:**

```
Input: root = [10,5,-3,3,2,null,11,3,-2,null,1], targetSum = 8
Output: 3
Explanation: The paths that sum to 8 are shown.
```

**Example 2:**

```
Input: root = [5,4,8,11,null,13,4,7,2,null,null,5,1], targetSum = 22
Output: 3
```

 

**Constraints:**

- The number of nodes in the tree is in the range `[0, 1000]`.
- `-10^9 <= Node.val <= 10^9`
- `-1000 <= targetSum <= 1000`

## Solution

```python
from typing import Optional


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        def dfs(node, cur_sum):
            if not node:
                return
            nonlocal ans
            cur_sum += node.val
            old_sum = cur_sum - targetSum
            ans += sum_map.get(old_sum, 0)
            sum_map[cur_sum] = sum_map.get(cur_sum, 0) + 1
            dfs(node.left, cur_sum)
            dfs(node.right, cur_sum)
            sum_map[cur_sum] -= 1

        if not root:
            return 0
        ans = 0
        sum_map = {0: 1}
        dfs(root, 0)
        return ans
```


## Explanation

### Approach: Prefix Sum Technique

This solution uses **prefix sums** to efficiently solve array problems by precomputing cumulative sums.

### Intuition

As we walk from the root to the current node, we keep the running sum of that path. If an earlier prefix sum was `current_sum - targetSum`, then the nodes after that earlier point form a downward path whose total is exactly `targetSum`. A hash map lets us count how many such prefixes exist in constant time.

### Algorithm

Here's the step-by-step approach:

1. Traverse the tree with DFS while maintaining the running path sum.
2. At each node, add its value to the running sum and look up how many times `current_sum - targetSum` has appeared before.
3. Store the current prefix sum in the map, recurse into the left and right children, then remove the current sum when backtracking.
4. Accumulate the count of valid paths and return the final total.

### Code Breakdown

`sum_map` records how many times each prefix sum appears along the current root-to-node path. The backtracking step `sum_map[cur_sum] -= 1` is crucial because prefix sums from one branch must not leak into sibling branches.

### Why This Works

- **Prefix-sum identity:** A path sum equals `targetSum` exactly when two prefix sums differ by `targetSum`.
- **Path-local bookkeeping:** The map only contains sums on the current DFS path, so every counted path is downward and valid.
- **Backtracking correctness:** Removing the current prefix sum after exploring children restores the correct state for the parent call.
- **Optimization:** Balances time complexity gains with reasonable space usage


## Complexity

- **Time**: `O(n)`
- **Space**: `O(n)`
