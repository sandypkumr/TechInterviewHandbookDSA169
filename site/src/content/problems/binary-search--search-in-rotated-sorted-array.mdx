## Problem Statement

There is an integer array `nums` sorted in ascending order (with **distinct** values).

Prior to being passed to your function, `nums` is **possibly left rotated** at an unknown index `k` (`1 <= k < nums.length`) such that the resulting array is `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]` (**0-indexed**). For example, `[0,1,2,4,5,6,7]` might be left rotated by `3` indices and become `[4,5,6,7,0,1,2]`.

Given the array `nums` **after** the possible rotation and an integer `target`, return _the index of _`target`_ if it is in _`nums`_, or _`-1`_ if it is not in _`nums`.

You must write an algorithm with `O(log n)` runtime complexity.

 

**Example 1:**

```
Input: nums = [4,5,6,7,0,1,2], target = 0
Output: 4
```
**Example 2:**

```
Input: nums = [4,5,6,7,0,1,2], target = 3
Output: -1
```
**Example 3:**

```
Input: nums = [1], target = 0
Output: -1
```

 

**Constraints:**

- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i] <= 10^4`
- All values of `nums` are **unique**.
- `nums` is an ascending array that is possibly rotated.
- `-10^4 <= target <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                return mid
            if nums[mid] >= nums[left]:
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1
        return -1
```


## Explanation

### Approach: Binary Search with Sorted-Half Detection

Even after rotation, at least one side of `mid` is always sorted.
At each step:
1. Find `mid`
2. Determine which half is sorted
3. Check whether `target` lies in that sorted half
4. Discard the other half

So we still eliminate half of the array each iteration.

### Key Observation

If `nums[mid] >= nums[left]`, then left half `[left..mid]` is sorted.
Otherwise, right half `[mid..right]` is sorted.

### Step-by-Step Example

Input: `nums = [4,5,6,7,0,1,2]`, `target = 0`

- `left=0, right=6, mid=3 (value 7)`
  - left half `[4,5,6,7]` sorted
  - target 0 not in [4..7], move right side: `left = mid + 1 = 4`

- `left=4, right=6, mid=5 (value 1)`
  - left half `[0,1]` sorted
  - target 0 in [0..1], move left side: `right = mid - 1 = 4`

- `left=4, right=4, mid=4 (value 0)` -> found

### Code Breakdown

```python
if nums[mid] >= nums[left]:
    # left half sorted
    if nums[left] <= target < nums[mid]:
        right = mid - 1
    else:
        left = mid + 1
else:
    # right half sorted
    if nums[mid] < target <= nums[right]:
        left = mid + 1
    else:
        right = mid - 1
```

### Why This Works

- Distinct values guarantee sorted-half checks are unambiguous
- Each loop removes half of remaining search space
- Therefore runtime is O(log n), as required
