## Problem Statement

You are a product manager and currently leading a team to develop a new product. Unfortunately, the latest version of your product fails the quality check. Since each version is developed based on the previous version, all the versions after a bad version are also bad.

Suppose you have `n` versions `[1, 2, ..., n]` and you want to find out the first bad one, which causes all the following ones to be bad.

You are given an API `bool isBadVersion(version)` which returns whether `version` is bad. Implement a function to find the first bad version. You should minimize the number of calls to the API.

 

**Example 1:**

```
Input: n = 5, bad = 4
Output: 4
Explanation:
call isBadVersion(3) -> false
call isBadVersion(5) -> true
call isBadVersion(4) -> true
Then 4 is the first bad version.
```

**Example 2:**

```
Input: n = 1, bad = 1
Output: 1
```

 

**Constraints:**

- `1 <= bad <= n <= 2^31 - 1`

```python
class Solution:
    def firstBadVersion(self, n: int) -> int:
        left, right = 1, n
        while left < right:
            mid = left + (right - left) // 2
            if isBadVersion(mid):
                right = mid
            else:
                left = mid + 1
        return left
```


## Explanation

### Approach

This solution uses **Depth-First Search (DFS)** to explore all possible paths. It uses the **Two Pointers** technique to efficiently search through the data. It applies **Binary Search** to find the target in logarithmic time.

### Intuition

By maintaining a sorted search space, we can eliminate half of the remaining candidates with each comparison, leading to logarithmic time complexity.

### Algorithm

1. **Main loop**: Continue until condition is met
2. **Move pointers**: Adjust left/right pointers based on current sum
3. **Return result**: Output the computed answer

### Why This Works

- **Two pointers optimization**: Eliminates need for nested loops by intelligently moving pointers
- **Logarithmic search**: Halves search space with each iteration
- **Correctness**: The algorithm handles all edge cases and produces the expected output
- **Efficiency**: Optimized for both time and space constraints
