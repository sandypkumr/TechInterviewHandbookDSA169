## Problem Statement

Given a list of non-negative integers `nums`, arrange them such that they form the largest number and return it.

Since the result may be very large, so you need to return a string instead of an integer.

 

**Example 1:**

```
Input: nums = [10,2]
Output: "210"
```

**Example 2:**

```
Input: nums = [3,30,34,5,9]
Output: "9534330"
```

 

**Constraints:**

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 10^9`

## Solution

```python
from typing import List


class Solution:
    def largestNumber(self, nums: List[int]) -> str:
        nums = [str(num) for num in nums]
        nums.sort(key=lambda x: x * 10, reverse=True)
        return str(int(''.join(nums)))
```


## Explanation

### Approach: Greedy Algorithm

This solution uses a **Greedy** strategy to make locally optimal decisions that lead to the global optimum.

### Intuition

The best order is not determined by the numeric value of each element alone. For two numbers `x` and `y`, the right comparison is whether `xy` or `yx` produces the larger concatenated string. Sorting by that idea makes the full joined result as large as possible.

### Algorithm

Here's the step-by-step approach:

1. Convert every number to a string.
2. Sort the strings so the concatenation order favors larger combined prefixes.
3. Join the sorted strings into one result.
4. Convert through `int(...)` and back to `str(...)` to collapse cases like `"00"` into `"0"`.

### Code Breakdown

The key line is the custom sort key `x * 10`, which works here because the input bounds keep the repeated string long enough to compare concatenation order reliably. After sorting in descending order, joining the pieces produces the largest possible number string.

### Why This Works

- **Pairwise optimal ordering:** If `xy >= yx`, placing `x` before `y` cannot hurt the final result.
- **Global consistency:** Sorting all strings by that comparison yields the lexicographically largest concatenation.
- **Leading-zero fix:** Converting through `int` ensures that all-zero inputs return exactly `"0"`.


## Complexity

- **Time**: `O(n log n)`
- **Space**: `O(n)`
