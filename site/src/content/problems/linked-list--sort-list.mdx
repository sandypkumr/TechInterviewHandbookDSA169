## Problem Statement

Given the `head` of a linked list, return _the list after sorting it in **ascending order**_.

 

**Example 1:**

```
Input: head = [4,2,1,3]
Output: [1,2,3,4]
```

**Example 2:**

```
Input: head = [-1,5,3,4,0]
Output: [-1,0,3,4,5]
```

**Example 3:**

```
Input: head = []
Output: []
```

 

**Constraints:**

- The number of nodes in the list is in the range `[0, 5 * 10^4]`.
- `-10^5 <= Node.val <= 10^5`

 

**Follow up:** Can you sort the linked list in `O(n logn)` time and `O(1)` memory (i.e. constant space)?

## Solution

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def sortList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        if not head or not head.next:
            return head
        slow = head
        fast = head.next
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        mid = slow.next
        slow.next = None
        return self.merge(self.sortList(head), self.sortList(mid))

    def merge(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode()
        current = dummy
        while list1 and list2:
            if list1.val < list2.val:
                current.next = list1
                list1 = list1.next
            else:
                current.next = list2
                list2 = list2.next
            current = current.next
        current.next = list1 if list1 else list2
        return dummy.next
```


## Explanation

### Approach: Merge Sort on Linked List

Linked lists are ideal for merge sort:

1. Split list into two halves (fast/slow pointers)
2. Recursively sort each half
3. Merge two sorted halves

### Split Logic

`fast` moves two steps, `slow` one step. When `fast` ends, `slow` is near midpoint.
Then cut at `slow.next`.

### Merge Logic

Use dummy head and repeatedly pick smaller front node from two lists.

### Why This Works

- Divide-and-conquer guarantees sorted merge of sorted halves
- Every level touches all nodes once, with `log n` levels
- Time O(n log n), recursion stack O(log n)
