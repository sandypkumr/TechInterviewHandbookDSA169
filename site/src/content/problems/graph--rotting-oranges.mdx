## Problem Statement

You are given an `m x n` `grid` where each cell can have one of three values:

- `0` representing an empty cell,
- `1` representing a fresh orange, or
- `2` representing a rotten orange.

Every minute, any fresh orange that is **4-directionally adjacent** to a rotten orange becomes rotten.

Return _the minimum number of minutes that must elapse until no cell has a fresh orange_. If _this is impossible, return_ `-1`.

 

**Example 1:**

```
Input: grid = [[2,1,1],[1,1,0],[0,1,1]]
Output: 4
```

**Example 2:**

```
Input: grid = [[2,1,1],[0,1,1],[1,0,1]]
Output: -1
Explanation: The orange in the bottom left corner (row 2, column 0) is never rotten, because rotting only happens 4-directionally.
```

**Example 3:**

```
Input: grid = [[0,2]]
Output: 0
Explanation: Since there are already no fresh oranges at minute 0, the answer is just 0.
```

 

**Constraints:**

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 10`
- `grid[i][j]` is `0`, `1`, or `2`.

## Solution

```python
from typing import List


class Solution:
    def orangesRotting(self, grid: List[List[int]]) -> int:
        if not grid:
            return 0
        queue = []
        fresh = 0
        for i in range(len(grid)):
            for j in range(len(grid[0])):
                if grid[i][j] == 2:
                    queue.append((i, j))
                elif grid[i][j] == 1:
                    fresh += 1
        if fresh == 0:
            return 0
        minutes = 0
        directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]
        while queue and fresh > 0:
            minutes += 1
            for _ in range(len(queue)):
                x, y = queue.pop(0)
                for dx, dy in directions:
                    nx, ny = x + dx, y + dy
                    if 0 <= nx < len(grid) and 0 <= ny < len(grid[0]) and grid[nx][ny] == 1:
                        grid[nx][ny] = 2
                        fresh -= 1
                        queue.append((nx, ny))
        return minutes if fresh == 0 else -1
```


## Explanation

### Approach: Multi-Source BFS

This solution uses **multi-source BFS** starting from every rotten orange, because rot spreads outward one minute at a time.

### Intuition

All initially rotten oranges begin spreading simultaneously. That matches BFS perfectly: each level of the search represents one minute, and any fresh orange reached in that level becomes rotten for the next minute.

### Algorithm

Here's the step-by-step approach:

1. Scan the grid to collect all initially rotten oranges and count the fresh ones.
2. If there are no fresh oranges, return `0` immediately.
3. Run BFS level by level, rotting all fresh neighbors of the current rotten oranges and decreasing the fresh count.
4. Return the minutes used if all fresh oranges rot; otherwise return `-1`.

### Code Breakdown

The queue stores the oranges that are currently rotten. Each outer-loop iteration represents one minute, and the inner loop processes exactly the oranges that can spread rot during that minute.

### Why This Works

- **Simultaneous spread:** Multi-source BFS models all rotten oranges infecting neighbors at the same time.
- **Minute-by-minute layering:** Each BFS level corresponds to one minute elapsed.
- **Fresh count validation:** If fresh oranges remain after the queue is exhausted, they were unreachable and the answer must be `-1`.
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage

- **Time**: `O(m×n)`
- **Space**: `O(m×n)`
