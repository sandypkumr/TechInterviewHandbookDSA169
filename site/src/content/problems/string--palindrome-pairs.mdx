## Problem Statement

You are given a **0-indexed** array of **unique** strings `words`.

A **palindrome pair** is a pair of integers `(i, j)` such that:

- `0 <= i, j < words.length`,
- `i != j`, and
- `words[i] + words[j]` (the concatenation of the two strings) is a palindrome.

Return _an array of all the **palindrome pairs** of _`words`.

You must write an algorithm with `O(sum of words[i].length)` runtime complexity.

 

**Example 1:**

```
Input: words = ["abcd","dcba","lls","s","sssll"]
Output: [[0,1],[1,0],[3,2],[2,4]]
Explanation: The palindromes are ["abcddcba","dcbaabcd","slls","llssssll"]
```

**Example 2:**

```
Input: words = ["bat","tab","cat"]
Output: [[0,1],[1,0]]
Explanation: The palindromes are ["battab","tabbat"]
```

**Example 3:**

```
Input: words = ["a",""]
Output: [[0,1],[1,0]]
Explanation: The palindromes are ["a","a"]
```

 

**Constraints:**

- `1 <= words.length <= 5000`
- `0 <= words[i].length <= 300`
- `words[i]` consists of lowercase English letters.

## Solution

```python
from typing import List


class Solution:
    def palindromePairs(self, words: List[str]) -> List[List[int]]:
        def is_palindrome(s : str) -> bool:
            return s == s[::-1]

        word_to_index = {word: i for i, word in enumerate(words)}
        result = []
        for i, word in enumerate(words):
            for j in range(len(word) + 1):
                prefix, suffix = word[:j], word[j:]
                if is_palindrome(prefix):
                    reversed_suffix = suffix[::-1]
                    if reversed_suffix in word_to_index and word_to_index[reversed_suffix] != i:
                        result.append([word_to_index[reversed_suffix], i])
                if j != len(word) and is_palindrome(suffix):
                    reversed_prefix = prefix[::-1]
                    if reversed_prefix in word_to_index and word_to_index[reversed_prefix] != i:
                        result.append([i, word_to_index[reversed_prefix]])
        return result
```


## Explanation

### Approach: Hash Map Lookup

This solution stores each word's index in a **hash map** and checks every possible split of every word.

### Intuition

Suppose we split a word into `prefix + suffix`. If `prefix` is already a palindrome, then we only need some other word equal to `reverse(suffix)` to place in front. Symmetrically, if `suffix` is a palindrome, then `reverse(prefix)` can be appended after the current word.

### Algorithm

Here's the step-by-step approach:

1. Build a map from each word to its index.
2. For every word, try every split position `j` from `0` to `len(word)`.
3. If the prefix is a palindrome, look for `reverse(suffix)` in the map; if the suffix is a palindrome, look for `reverse(prefix)`.
4. Add the matching index pairs whenever the partner word exists and is not the same word.

### Code Breakdown

The helper `is_palindrome` checks whether a prefix or suffix can serve as the mirrored half around another word. `word_to_index` makes reverse-word lookups constant time, which keeps the full search practical.

### Why This Works

- **Complete split coverage:** Every palindrome pair must align with some split where one side of a word is itself a palindrome.
- **Correct reverse lookup:** The non-palindromic side must be mirrored by the reverse of another word.
- **No self-pair mistakes:** The index check prevents pairing a word with itself unless a distinct matching word exists.


## Complexity

- **Time**: `O(n k²)`
- **Space**: `O(n k)`
