## Problem Statement

Given the `root` of a binary tree, return _the zigzag level order traversal of its nodes' values_. (i.e., from left to right, then right to left for the next level and alternate between).

 

**Example 1:**

```
Input: root = [3,9,20,null,null,15,7]
Output: [[3],[20,9],[15,7]]
```

**Example 2:**

```
Input: root = [1]
Output: [[1]]
```

**Example 3:**

```
Input: root = []
Output: []
```

 

**Constraints:**

- The number of nodes in the tree is in the range `[0, 2000]`.
- `-100 <= Node.val <= 100`

## Solution

```python
from collections import deque
from typing import Optional, List


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def zigzagLevelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        res = []
        q = deque([root])
        level = 0
        while q:
            level += 1
            new_q = deque()
            level_res = []
            for node in q:
                level_res.append(node.val)
                if node.left:
                    new_q.append(node.left)
                if node.right:
                    new_q.append(node.right)
            if level % 2 == 0:
                level_res = level_res[::-1]
            res.append(level_res)
            q = new_q
        return res
```


## Explanation

### Approach: BFS

This solution uses **Breadth-First Search (BFS)** with a queue to explore states level by level.

### Intuition

We still want a normal level-order traversal, but the output direction alternates at each depth. BFS already groups nodes by level, so the only extra work is to reverse the values on every other level before appending them.

### Algorithm

Here's the step-by-step approach:

1. Return an empty list if the tree is empty.
2. Use a queue to process one level at a time.
3. Collect the node values for the current level while building the queue for the next level.
4. Reverse the collected values on even-numbered levels, append them to the result, and continue.

### Code Breakdown

The variable `level` tracks whether the current depth should be read left-to-right or right-to-left. The code stores the next layer in `new_q`, so each outer-loop iteration works on exactly one tree level

### Why This Works

- **BFS preserves levels:** Every pass through the outer loop handles a single tree depth.
- **Alternating order:** Reversing `level_res` on every other level produces the zigzag pattern without changing traversal correctness.
- **Children still expand normally:** Nodes are always explored left child first, then right child, so the tree structure is processed consistently.

### Why This Works

- **Level-by-level exploration:** Guarantees we find the shortest path (if it's a path problem) and systematically covers all nodes
- **No nested loops:** Two pointers avoid O(n²) complexity by moving intelligently through the data
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
