## Problem Statement

There is a new alien language which uses the latin alphabet. However, the order among letters are unknown to you. You
receive a list of non-empty words from the dictionary, where words are sorted lexicographically by the rules of this
new language. Derive the order of letters in this language.


## Examples

For example, given the following words:

    [
    "wrt",
    "wrf",
    "er",
    "ett",
    "rftt"
    ]

The correct order is: "wertf".


## Constraints

Note:
1. You may assume all letters are in lowercase.
2. If the order is invalid, return an empty string.
3. There may be multiple valid order of letters, return any one of them is fine.


## Solution

```python
from collections import defaultdict
from typing import List

class Solution:
    def alienOrder(self, words: list[str]) -> str:
        graph = defaultdict(set)
        in_degree = {char: 0 for word in words for char in word}
        for i in range(len(words) - 1):
            word1, word2 = words[i], words[i + 1]
            if len(word1) > len(word2) and word1.startswith(word2):
                return ''
            if len(word1) < len(word2) and word2.startswith(word1):
                continue
            for c1, c2 in zip(word1, word2):
                if c1 != c2:
                    if c2 not in graph[c1]:
                        graph[c1].add(c2)
                        in_degree[c2] += 1
                    break

        queue = [char for char in in_degree if in_degree[char] == 0]
        result = []
        while queue:
            char = queue.pop(0)
            result.append(char)
            for next_char in graph[char]:
                in_degree[next_char] -= 1
                if in_degree[next_char] == 0:
                    queue.append(next_char)

        return ''.join(result) if len(result) == len(in_degree) else ''
```


## Explanation

### Approach: Build Graph + Kahn's Topological Sort

Treat each character as a node in a directed graph. When two adjacent words first differ at characters `c1` and `c2`, we know `c1` must come before `c2`, so we add an edge `c1 -> c2`. Once all ordering rules are collected, run topological sort to produce a valid alphabet order.

### Intuition

The given word list is already sorted in the alien language. That means every adjacent pair of words gives us local ordering information. The **first differing character** is the only one that matters, because later characters do not affect lexicographic order once an earlier difference is found.

### Algorithm

1. Initialize `in_degree` for every character that appears in the input
2. Compare each neighboring pair of words
3. If a longer word appears before its own prefix, return `""` because the order is invalid
4. Find the first differing characters and add a directed edge if it is new
5. Push all zero-indegree characters into a queue
6. Repeatedly pop from the queue, append to the answer, and reduce indegrees of neighbors
7. If not all characters are emitted, a cycle exists, so return `""`

### Code Breakdown

- `graph = defaultdict(set)`: adjacency list for precedence rules
- `in_degree = {char: 0 ...}`: counts how many prerequisites each character has
- `if len(word1) > len(word2) and word1.startswith(word2)`: catches invalid prefix ordering
- `if c2 not in graph[c1]`: avoids double-counting the same edge
- `queue = [char for char in in_degree if in_degree[char] == 0]`: starting characters with no prerequisites
- `return ''.join(result) if len(result) == len(in_degree) else ''`: detects cycles

### Why This Works

- Adjacent words capture all necessary ordering constraints
- Topological sort returns an order that respects every directed edge
- Invalid prefix cases are rejected immediately
- Cycles prevent a full topological ordering, so returning `""` is correct
