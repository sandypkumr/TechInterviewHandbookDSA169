## Problem Statement

Given an array `nums` of size `n`, return _the majority element_.

The majority element is the element that appears more than `&lfloor;n / 2&rfloor;` times. You may assume that the majority element always exists in the array.

 

**Example 1:**

```
Input: nums = [3,2,3]
Output: 3
```
**Example 2:**

```
Input: nums = [2,2,1,1,1,2,2]
Output: 2
```

 

**Constraints:**

- `n == nums.length`
- `1 <= n <= 5 * 10^4`
- `-10^9 <= nums[i] <= 10^9`
- The input is generated such that a majority element will exist in the array.

 

**Follow-up:** Could you solve the problem in linear time and in `O(1)` space?

## Solution

```python
from typing import List


class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        count = 0
        candidate = None
        for num in nums:
            if count == 0:
                candidate = num
            count += 1 if num == candidate else -1
        return candidate
```


## Explanation

### Approach: Boyer-Moore Voting (Pair Cancellation)

This algorithm keeps a current `candidate` and a `count`.
- Same value as candidate -> `count += 1`
- Different value -> `count -= 1`
- If `count == 0`, choose current number as new candidate

### Why Cancellation Works

Think of pairing one majority element with one non-majority element and removing both.
Because majority appears more than `n/2` times, it cannot be fully canceled. It will survive as the final candidate.

### Step-by-Step Walkthrough

**Input:** `[2,2,1,1,1,2,2]`

```
start: candidate=None, count=0

2: count==0 -> candidate=2, then match -> count=1
2: match candidate -> count=2
1: different -> count=1
1: different -> count=0
1: count==0 -> candidate=1, then match -> count=1
2: different -> count=0
2: count==0 -> candidate=2, then match -> count=1
```

Final candidate = `2`

### Code Breakdown

```python
if count == 0:
    candidate = num
count += 1 if num == candidate else -1
```

These two lines are the core of the algorithm.

### Important Note

In some variants where majority is not guaranteed, a second pass is needed to verify the candidate count. In this problem, majority is guaranteed by constraints, so one pass is enough.

### Why This is Optimal

- Time: O(n), one scan
- Space: O(1), only two variables
- Meets the follow-up requirement exactly
