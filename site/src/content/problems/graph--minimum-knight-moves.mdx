## Problem Statement

In an infinite chessboard, a knight is placed at the position (0, 0) and you are given a target position (x, y). Your
task is to determine the minimum number of moves required for the knight to reach the target position.

The knight moves in an "L" shape:

Two squares in one direction and one square in a perpendicular direction.
Or one square in one direction and two squares in a perpendicular direction.
You can assume the target position (x, y) is on the board, and both x and y are integers.


## Examples

Example 1:

Example 1:
Input: x = 2, y = 1
Output: 1

Example 2:

Example 2:
Input: x = 5, y = 5
Output: 4


## Solution

```python
from collections import deque


class Solution:
    def minKnightMoves(self, x: int, y: int) -> int:
        x, y = abs(x), abs(y)
        directions = [(1, 2), (2, 1), (2, -1), (1, -2), (-1, -2), (-2, -1), (-2, 1), (-1, 2)]
        queue = deque([(0, 0, 0)])
        visited = {(0, 0)}
        while queue:
            i, j, steps = queue.popleft()
            if i == x and j == y:
                return steps
            for di, dj in directions:
                ni, nj = i + di, j + dj
                if (ni, nj) not in visited and -2 <= ni <= x + 2 and -2 <= nj <= y + 2:
                    visited.add((ni, nj))
                    queue.append((ni, nj, steps + 1))
        return -1
```


## Explanation

### Approach: BFS

This solution uses **Breadth-First Search (BFS)** with a queue to explore states level by level.

### Intuition

Each knight move has the same cost, so the first time BFS reaches the target must be the minimum number of moves. The code also uses symmetry by reflecting the target into the first quadrant, which reduces the search space without changing the answer.

### Algorithm

Here's the step-by-step approach:

1. Convert `(x, y)` to `(abs(x), abs(y))` so the search only needs one symmetric region.
2. Start BFS from `(0, 0, 0)`, where the third value is the number of moves taken.
3. Try all eight knight directions, enqueueing valid unvisited positions inside a small bounding box around the target.
4. Return `steps` when the target position is dequeued.

### Code Breakdown

The `directions` list encodes the eight legal knight moves. The bounds `-2 <= ni <= x + 2` and `-2 <= nj <= y + 2` keep the search finite while still allowing the small detours a knight may need near the target.

### Why This Works

- **Uniform edge cost:** BFS guarantees the first visit to the target uses the fewest moves.
- **Symmetry preservation:** Reflecting coordinates into the first quadrant does not change the minimum distance for a knight.
- **Safe bounding:** The limited search box is wide enough to include optimal paths while preventing unbounded exploration.
