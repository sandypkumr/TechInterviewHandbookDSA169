## Problem Statement

Given an integer array `nums`, return `true` _if you can partition the array into two subsets such that the sum of the elements in both subsets is equal or _`false`_ otherwise_.

 

**Example 1:**

```
Input: nums = [1,5,11,5]
Output: true
Explanation: The array can be partitioned as [1, 5, 5] and [11].
```

**Example 2:**

```
Input: nums = [1,2,3,5]
Output: false
Explanation: The array cannot be partitioned into equal sum subsets.
```

 

**Constraints:**

- `1 <= nums.length <= 200`
- `1 <= nums[i] <= 100`

## Solution

```python
from typing import List


class Solution:
    def canPartition(self, nums: List[int]) -> bool:
        total = sum(nums)
        if total % 2 != 0:
            return False
        target = total // 2
        dp = [False] * (target + 1)
        dp[0] = True
        for num in nums:
            for i in range(target, num - 1, -1):
                dp[i] = dp[i] or dp[i - num]
        return dp[target]
```


## Explanation

### Approach: Dynamic Programming

This solution uses **Dynamic Programming** to build answers from smaller subproblems and avoid recomputation.

### Intuition

If the array can be split into two equal subsets, then one subset must sum to exactly half of the total. That turns the problem into a classic subset-sum question: can we pick some numbers that add up to `total // 2`? A 1D DP array is enough because each number is used at most once.

### Algorithm

Here's the step-by-step approach:

1. Compute the total sum. If it is odd, return `False` immediately because two equal subsets are impossible.
2. Let `target = total // 2` and create a boolean DP array where `dp[i]` means sum `i` is reachable.
3. Set `dp[0] = True`, then process each number by iterating backward from `target` down to that number.
4. Update `dp[i] = dp[i] or dp[i - num]` and return `dp[target]` at the end.

### Code Breakdown

The backward iteration is the key detail: it prevents the current number from being reused multiple times in the same pass. Each update asks whether the current sum `i` was already reachable or can be reached by adding `num` to a previously reachable sum `i - num`.

### Why This Works

- **Equivalent reformulation:** Partitioning into two equal subsets is exactly the same as finding one subset with sum `total // 2`.
- **Correct state meaning:** `dp[i]` records whether some subset of processed numbers can make sum `i`.
- **Single-use guarantee:** Iterating backward ensures each number contributes at most once, matching the problem constraints.


## Complexity

- **Time**: `O(n × sum)`
- **Space**: `O(sum)`
