## Problem Statement

You have `k` lists of sorted integers in **non-decreasing order**. Find the **smallest** range that includes at least one number from each of the `k` lists.

We define the range `[a, b]` is smaller than range `[c, d]` if `b - a < d - c` **or** `a < c` if `b - a == d - c`.

 

**Example 1:**

```
Input: nums = [[4,10,15,24,26],[0,9,12,20],[5,18,22,30]]
Output: [20,24]
Explanation: 
List 1: [4, 10, 15, 24,26], 24 is in range [20,24].
List 2: [0, 9, 12, 20], 20 is in range [20,24].
List 3: [5, 18, 22, 30], 22 is in range [20,24].
```

**Example 2:**

```
Input: nums = [[1,2,3],[1,2,3],[1,2,3]]
Output: [1,1]
```

 

**Constraints:**

- `nums.length == k`
- `1 <= k <= 3500`
- `1 <= nums[i].length <= 50`
- `-10^5 <= nums[i][j] <= 10^5`
- `nums[i]` is sorted in **non-decreasing** order.

## Solution

```python
import heapq
from typing import List


class Solution:
    def smallestRange(self, nums: List[List[int]]) -> List[int]:
        heap = []
        max_val = float('-inf')
        for i, num in enumerate(nums):
            heap.append((num[0], i, 0))
            max_val = max(max_val, num[0])
        heapq.heapify(heap)
        start, end = float('-inf'), float('inf')
        while len(heap) == len(nums):
            val, i, j = heapq.heappop(heap)
            if max_val - val < end - start:
                start, end = val, max_val
            if j + 1 < len(nums[i]):
                heapq.heappush(heap, (nums[i][j + 1], i, j + 1))
                max_val = max(max_val, nums[i][j + 1])
        return [start, end]
```


## Explanation

### Approach: Min-Heap

This solution keeps one active element from each list in a **min-heap** and tracks the current maximum among them.

### Intuition

At any moment, a valid range must include one chosen element from every list. If we always know the current minimum and maximum among those chosen elements, then we can evaluate the range they form. The only way to potentially improve the range is to advance the list that currently contributes the minimum value.

### Algorithm

Here's the step-by-step approach:

1. Put the first element from each list into a min-heap and record the maximum of those elements.
2. Repeatedly pop the smallest element from the heap and compare the current range `[min_val, max_val]` against the best answer so far.
3. Advance within the list that provided the minimum element, pushing its next value into the heap and updating `max_val`.
4. Stop when any list runs out, because a complete range using all lists is no longer possible.

### Code Breakdown

Each heap entry stores `(value, list_index, element_index)`, so after popping the minimum, the code knows exactly which list to advance. The variable `max_val` tracks the right edge of the current candidate range.

### Why This Works

- **One-from-each-list invariant:** The heap always contains one current candidate from every list while the loop runs.
- **Greedy advancement:** Moving the list with the current minimum is the only way to possibly shrink the range.
- **Exhaustion condition:** Once a list has no next element, no future range can cover all lists, so the best range seen is final.
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
