## Problem Statement

Suppose an array of length `n` sorted in ascending order is **rotated** between `1` and `n` times. For example, the array `nums = [0,1,2,4,5,6,7]` might become:

- `[4,5,6,7,0,1,2]` if it was rotated `4` times.
- `[0,1,2,4,5,6,7]` if it was rotated `7` times.

Notice that **rotating** an array `[a[0], a[1], a[2], ..., a[n-1]]` 1 time results in the array `[a[n-1], a[0], a[1], a[2], ..., a[n-2]]`.

Given the sorted rotated array `nums` of **unique** elements, return _the minimum element of this array_.

You must write an algorithm that runs in `O(log n) time`.

 

**Example 1:**

```
Input: nums = [3,4,5,1,2]
Output: 1
Explanation: The original array was [1,2,3,4,5] rotated 3 times.
```

**Example 2:**

```
Input: nums = [4,5,6,7,0,1,2]
Output: 0
Explanation: The original array was [0,1,2,4,5,6,7] and it was rotated 4 times.
```

**Example 3:**

```
Input: nums = [11,13,15,17]
Output: 11
Explanation: The original array was [11,13,15,17] and it was rotated 4 times.
```

 

**Constraints:**

- `n == nums.length`
- `1 <= n <= 5000`
- `-5000 <= nums[i] <= 5000`
- All the integers of `nums` are **unique**.
- `nums` is sorted and rotated between `1` and `n` times.

## Solution

```python
from typing import List


class Solution:
    def findMin(self, nums: List[int]) -> int:
        left, right = 0, len(nums) - 1
        while left < right:
            mid = (left + right) // 2
            if nums[mid] < nums[right]:
                right = mid
            else:
                left = mid + 1
        return nums[left]
```


## Explanation

### Approach: Binary Search on the Pivot Side

In a rotated sorted array, the minimum is the pivot point where order breaks.
We use binary search with `mid` vs `right` comparison to decide which side contains the minimum.

### Key Observation

- If `nums[mid] < nums[right]`, then right half `[mid..right]` is sorted and minimum is in `[left..mid]` (including `mid`).
- Else (`nums[mid] > nums[right]`), minimum must be in `[mid+1..right]`.

This lets us discard half the array each step.

### Step-by-Step Walkthrough

Input: `nums = [4,5,6,7,0,1,2]`

- `left=0, right=6, mid=3, nums[mid]=7, nums[right]=2`
  - `7 > 2` -> minimum is right side -> `left = 4`
- `left=4, right=6, mid=5, nums[mid]=1, nums[right]=2`
  - `1 < 2` -> minimum in left side incl mid -> `right = 5`
- `left=4, right=5, mid=4, nums[mid]=0, nums[right]=1`
  - `0 < 1` -> `right = 4`

Now `left == right == 4`, answer is `nums[4] = 0`.

### Code Breakdown

```python
while left < right:
    mid = (left + right) // 2
    if nums[mid] < nums[right]:

```

### Why This Works

- Invariant: minimum is always inside `[left, right]`
- Each iteration shrinks interval by about half
- Distinct values make comparison decisions unambiguous

- **Time**: `O(log n)`
- **Space**: `O(1)`
