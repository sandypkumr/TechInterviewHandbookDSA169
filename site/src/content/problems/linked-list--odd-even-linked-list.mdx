## Problem Statement

Given the `head` of a singly linked list, group all the nodes with odd indices together followed by the nodes with even indices, and return _the reordered list_.

The **first** node is considered **odd**, and the **second** node is **even**, and so on.

Note that the relative order inside both the even and odd groups should remain as it was in the input.

You must solve the problem in `O(1)` extra space complexity and `O(n)` time complexity.

 

**Example 1:**

```
Input: head = [1,2,3,4,5]
Output: [1,3,5,2,4]
```

**Example 2:**

```
Input: head = [2,1,3,5,6,4,7]
Output: [2,3,6,7,1,5,4]
```

 

**Constraints:**

- The number of nodes in the linked list is in the range `[0, 10^4]`.
- `-10^6 <= Node.val <= 10^6`

## Solution

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def oddEvenList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        if not head:
            return head
        odd = head
        even = head.next
        even_head = even
        while even and even.next:
            odd.next = even.next
            odd = odd.next
            even.next = odd.next
            even = even.next
        odd.next = even_head
        return head
```


## Explanation

### Approach: In-place Odd/Even Chain Split and Stitch

Maintain two running chains:
- odd-indexed nodes via `odd`
- even-indexed nodes via `even`

Store `even_head` so we can append the even chain after odd chain at the end.

### Step-by-Step Example

Input: `1 -> 2 -> 3 -> 4 -> 5`

- start: `odd=1`, `even=2`, `even_head=2`
- link odd to next odd: `1 -> 3`, move `odd=3`
- link even to next even: `2 -> 4`, move `even=4`
- next round: `3 -> 5`, even chain unchanged (`4` has no even-next)
- stitch: `5.next = even_head (2)`

Result: `1 -> 3 -> 5 -> 2 -> 4`

Relative order within odd group and even group is preserved.

### Code Breakdown

```python
odd = head
even = head.next
even_head = even
while even and even.next:
    odd.next = even.next
    odd = odd.next
    even.next = odd.next
    even = even.next
odd.next = even_head
```

### Why This Works

- Each pointer advances through its own parity chain
- No new nodes are created; only `next` links are rewired
- One pass over list -> O(n) time, O(1) extra space

- **Time**: `O(n)`
- **Space**: `O(1)`
