## Problem Statement

Given an array of integers `nums` and an integer `target`, return _indices of the two numbers such that they add up to `target`_.

You may assume that each input would have **_exactly_ one solution**, and you may not use the _same_ element twice.

You can return the answer in any order.

 

**Example 1:**

```
Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: Because nums[0] + nums[1] == 9, we return [0, 1].
```

**Example 2:**

```
Input: nums = [3,2,4], target = 6
Output: [1,2]
```

**Example 3:**

```
Input: nums = [3,3], target = 6
Output: [0,1]
```

 

**Constraints:**

- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- **Only one valid answer exists.**

 

**Follow-up: **Can you come up with an algorithm that is less than `O(n^2)` time complexity?

## Solution

```python
from typing import List


class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        d = {}
        for i, num in enumerate(nums):
            if target - num in d:
                return [d[target - num], i]
            d[num] = i
        return []
```


## Explanation

### Approach: Hash Map - Store and Lookup Complements

The brilliant insight here is to use a **hash map to store complements**. As we iterate through the array, for each number we check: "Have I seen the number that would complete this pair?" If yes, we've found our answer.

### The Key Idea

Instead of brute force checking every pair (O(n²)), we ask a smarter question: *"What number do I need to reach the target?"*

For any number `num`, the complement is `target - num`. We store each number we've seen, so when we encounter a new number, we instantly know if its complement exists (O(1) lookup).

### Step-by-Step Walkthrough

**Input:** `nums = [2,7,11,15], target = 9`

```
i=0, num=2:
  Complement needed: 9 - 2 = 7
  Is 7 in map? NO
  Store: {2: 0}  (number 2 is at index 0)

i=1, num=7:
  Complement needed: 9 - 7 = 2
  Is 2 in map? YES! ✓
  Return: [map[2], current_index] = [0, 1]
```

The solution found the pair (2, 7) at indices (0, 1) that sum to 9!

### How the Code Works

```python
d = {}  # Hash map: number → index
for i, num in enumerate(nums):
    complement = target - num
    if complement in d:  # O(1) lookup!
        return [d[complement], i]
    d[num] = i  # Store this number for future checks
```

**Each line:**
1. `complement = target - num` - Calculate what we're looking for
2. `if complement in d:` - Check if we've already seen it (this is O(1)!)
3. `d[num] = i` - Store current number so future iterations can find it

### Why This is Optimal

**Time: O(n)** - Single pass, hash map operations are O(1)
**Space: O(n)** - Store up to n numbers in the map

### Comparison: Brute Force vs Hash Map

**Brute Force O(n²):**
```python
for i in range(n):
    for j in range(i+1, n):
        if nums[i] + nums[j] == target:
            return [i, j]
```
We check every possible pair - very slow!

**Hash Map O(n):**
We only pass through the array once, using the map to instantly find partners. This is why Two Sum is a classic interview question - it teaches the power of trading space for time!
