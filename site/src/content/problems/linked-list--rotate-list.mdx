## Problem Statement

Given the `head` of a linked list, rotate the list to the right by `k` places.

 

**Example 1:**

```
Input: head = [1,2,3,4,5], k = 2
Output: [4,5,1,2,3]
```

**Example 2:**

```
Input: head = [0,1,2], k = 4
Output: [2,0,1]
```

 

**Constraints:**

- The number of nodes in the list is in the range `[0, 500]`.
- `-100 <= Node.val <= 100`
- `0 <= k <= 2 * 10^9`

## Solution

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def rotateRight(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        if not head or not head.next:
            return head
        length = 1
        current = head
        while current.next:
            current = current.next
            length += 1
        current.next = head
        k %= length
        for _ in range(length - k):
            current = current.next
        head = current.next
        current.next = None
        return head
```


## Explanation

### Approach: Make Circular, Then Break at New Tail

Steps:
1. find list length and tail
2. connect tail to head (circular list)
3. normalize rotations: `k %= length`
4. move `length - k` steps from old tail to reach new tail
5. new head is `new_tail.next`; break cycle

### Why `k %= length`

Rotating by full list length returns original list, so only remainder matters.

### Code Core

```python
current.next = head   # make circular
k %= length
for _ in range(length - k):
    current = current.next
head = current.next
current.next = None
```

### Example

`[1,2,3,4,5]`, `k=2`
- circular: `...5->1`
- move `length-k = 3` from old tail (`5`) => lands on `3` (new tail)
- new head `4`, break after `3`
- result `[4,5,1,2,3]`

### Why This Works

- Circular connection converts rotation into finding a split point
- One pass for length + one pass to split
- O(n) time, O(1) extra space

- **Time**: `O(n)`
