## Problem Statement

You are given an `m x n` integer matrix `matrix` with the following two properties:

- Each row is sorted in non-decreasing order.
- The first integer of each row is greater than the last integer of the previous row.

Given an integer `target`, return `true` _if_ `target` _is in_ `matrix` _or_ `false` _otherwise_.

You must write a solution in `O(log(m * n))` time complexity.

 

**Example 1:**

```
Input: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 3
Output: true
```

**Example 2:**

```
Input: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 13
Output: false
```

 

**Constraints:**

- `m == matrix.length`
- `n == matrix[i].length`
- `1 <= m, n <= 100`
- `-10^4 <= matrix[i][j], target <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        if not matrix or not matrix[0]:
            return False
        m, n = len(matrix), len(matrix[0])
        left, right = 0, m * n - 1
        while left <= right:
            mid = (left + right) // 2
            if matrix[mid // n][mid % n] == target:
                return True
            elif matrix[mid // n][mid % n] < target:
                left = mid + 1
            else:
                right = mid - 1
        return False
```


## Explanation

### Approach: Binary Search on a Virtual 1D Array

Because:
- each row is sorted, and
- each row's first element is greater than previous row's last,

the whole matrix behaves like one sorted array of length `m*n`.

So we can binary search over indices `[0 .. m*n-1]`.

### Index Mapping Trick

For a virtual index `idx`:

- row = `idx // n`
- col = `idx % n`

So matrix value is `matrix[idx // n][idx % n]`.

### Step-by-Step Example

Input: `matrix=[[1,3,5,7],[10,11,16,20],[23,30,34,60]]`, `target=16`

- `m=3, n=4`, search range `0..11`
- `mid=5` -> `(1,1)` -> value `11 < 16`, go right
- `mid=8` -> `(2,0)` -> value `23 > 16`, go left
- `mid=6` -> `(1,2)` -> value `16` -> found

### Code Breakdown

```python
left, right = 0, m * n - 1
while left <= right:
    mid = (left + right) // 2
    val = matrix[mid // n][mid % n]
    if val == target:
        return True
    elif val < target:
        left = mid + 1
    else:
        right = mid - 1
```

### Why This Works

- Virtual 1D ordering is globally sorted
- Binary search discards half each step
- Uses O(1) extra space and meets required O(log(m*n)) time
