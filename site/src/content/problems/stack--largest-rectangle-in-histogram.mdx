## Problem Statement

Given an array of integers `heights` representing the histogram's bar height where the width of each bar is `1`, return _the area of the largest rectangle in the histogram_.

 

**Example 1:**

```
Input: heights = [2,1,5,6,2,3]
Output: 10
Explanation: The above is a histogram where width of each bar is 1.
The largest rectangle is shown in the red area, which has an area = 10 units.
```

**Example 2:**

```
Input: heights = [2,4]
Output: 4
```

 

**Constraints:**

- `1 <= heights.length <= 10^5`
- `0 <= heights[i] <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def largestRectangleArea(self, heights: List[int]) -> int:
        stack = []
        max_area = 0
        for i, h in enumerate(heights):
            while stack and h < heights[stack[-1]]:
                height = heights[stack.pop()]
                width = i if not stack else i - stack[-1] - 1
                max_area = max(max_area, height * width)
            stack.append(i)
        while stack:
            height = heights[stack.pop()]
            width = len(heights) if not stack else len(heights) - stack[-1] - 1
            max_area = max(max_area, height * width)
        return max_area
```


## Explanation

### Approach: Monotonic Stack

This solution uses a **monotonic increasing stack** of bar indices to determine how far each bar can extend as the limiting height of a rectangle.

### Intuition

When we see a bar shorter than the one on top of the stack, we know the taller bar's rectangle must end at the current index. Popping that bar reveals both boundaries of the widest rectangle for which that height is the minimum.

### Algorithm

Here's the step-by-step approach:

1. Scan the histogram from left to right while keeping indices of nondecreasing heights on the stack.
2. When a shorter bar appears, repeatedly pop taller bars and compute the rectangle area they can form.
3. Use the current index and the new stack top to determine each popped bar's width.
4. After the scan, pop any remaining bars and compute their final areas.

### Code Breakdown

The stack stores indices rather than heights so the code can compute widths precisely. Each index is pushed once and popped once, which makes the total work linear.

### Why This Works

- **Left and right boundaries:** A bar is popped exactly when its first smaller bar on the right is found, and the new stack top gives the first smaller bar on the left.
- **Maximal width per height:** That pair of boundaries defines the widest rectangle where the popped bar is the limiting height.
- **Complete coverage:** Every possible maximal rectangle is evaluated when its limiting bar is popped.
