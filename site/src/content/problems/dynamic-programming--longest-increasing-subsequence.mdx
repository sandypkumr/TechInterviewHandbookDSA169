## Problem Statement

Given an integer array `nums`, return _the length of the longest **strictly increasing **__**subsequence**_.

 

**Example 1:**

```
Input: nums = [10,9,2,5,3,7,101,18]
Output: 4
Explanation: The longest increasing subsequence is [2,3,7,101], therefore the length is 4.
```

**Example 2:**

```
Input: nums = [0,1,0,3,2,3]
Output: 4
```

**Example 3:**

```
Input: nums = [7,7,7,7,7,7,7]
Output: 1
```

 

**Constraints:**

- `1 <= nums.length <= 2500`
- `-10^4 <= nums[i] <= 10^4`

 

**Follow up:** Can you come up with an algorithm that runs in `O(n log(n))` time complexity?

## Solution

```python
from typing import List


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        if not nums:
            return 0
        dp = [1] * len(nums)
        for i in range(1, len(nums)):
            for j in range(i):
                if nums[i] > nums[j]:
                    dp[i] = max(dp[i], dp[j] + 1)
        return max(dp)
```


## Explanation

### Approach: Dynamic Programming

This solution uses **Dynamic Programming** to build answers from smaller subproblems and avoid recomputation.

### Intuition

Let `dp[i]` be the length of the longest increasing subsequence that **ends** at index `i`. Every element can at least form a subsequence of length `1` by itself. To extend a subsequence, look at every earlier index `j < i`; if `nums[j] < nums[i]`, then `nums[i]` can be appended to the subsequence ending at `j`.

### Algorithm

Here's the step-by-step approach:

1. Return `0` immediately if the array is empty.
2. Initialize `dp` with all `1`s, since each number is an increasing subsequence of length `1` on its own.
3. For each index `i`, scan all earlier indices `j`. If `nums[i] > nums[j]`, update `dp[i] = max(dp[i], dp[j] + 1)`.
4. Return the maximum value in `dp`, which is the best subsequence length ending anywhere in the array.

### Code Breakdown

The nested loops compare each element with all earlier elements. The inner loop computes the best increasing subsequence that can be extended by `nums[i]`, and the outer loop ensures those earlier answers are already known before they are used.

### Why This Works

- **Accurate state:** `dp[i]` captures exactly the best increasing subsequence ending at `i`.
- **Complete transition:** Checking every earlier `j` guarantees we consider every valid predecessor for `nums[i]`.
- **Global answer:** The longest subsequence may end at any index, so taking `max(dp)` gives the final result.


## Complexity

- **Time**: `O(n log n)`
- **Space**: `O(n)`
