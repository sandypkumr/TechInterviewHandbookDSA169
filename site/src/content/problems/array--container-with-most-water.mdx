## Problem Statement

You are given an integer array `height` of length `n`. There are `n` vertical lines drawn such that the two endpoints of the `i^th` line are `(i, 0)` and `(i, height[i])`.

Find two lines that together with the x-axis form a container, such that the container contains the most water.

Return _the maximum amount of water a container can store_.

**Notice** that you may not slant the container.

 

**Example 1:**

```
Input: height = [1,8,6,2,5,4,8,3,7]
Output: 49
Explanation: The above vertical lines are represented by array [1,8,6,2,5,4,8,3,7]. In this case, the max area of water (blue section) the container can contain is 49.
```

**Example 2:**

```
Input: height = [1,1]
Output: 1
```

 

**Constraints:**

- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def maxArea(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        res = 0
        while left < right:
            res = max(res, min(height[left], height[right]) * (right - left))
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        return res
```


## Explanation

### Approach: Two Pointers - Greedy Strategy

Start with pointers at both ends (maximum width), then move the pointer pointing to the **shorter bar** inward. This greedy choice works because: if we move the taller bar, we lose width and can't gain height. But moving the shorter bar gives us a chance to find a taller one.

### Why This Greedy Choice is Optimal

At any point, area = `min(height[left], height[right]) × width`. The minimum height is the bottleneck. Moving the taller bar can only decrease width - even if we find a taller bar, we'd need it to be significantly taller to compensate for lost width. Moving the shorter bar is the only way to potentially improve.

### Step-by-Step Walkthrough

**Input:** `height = [1,8,6,2,5,4,8,3,7]`

```
Initial: left=0 (h=1), right=8 (h=7)
Area = min(1,7) * 8 = 8
Shorter is left (1), move it →

left=1 (h=8), right=8 (h=7)
Area = min(8,7) * 7 = 49 ✓ NEW MAX!
Shorter is right (7), move it →

left=1 (h=8), right=7 (h=3)
Area = min(8,3) * 6 = 18 (less than 49)
Shorter is right (3), move it →

Continue until left >= right...
```

Maximum area: **49** at indices 1 and 8

### Code Breakdown

```python
left, right = 0, len(height) - 1
while left < right:
    # Calculate area: min height * width
    res = max(res, min(height[left], height[right]) * (right - left))

    # Move the pointer at the shorter bar
    if height[left] < height[right]:
        left += 1
    else:
        right -= 1
```

### Why O(n) is Optimal

Each pointer makes at most n moves, never backtracks. We guarantee checking the best configuration through the invariant that moving the taller bar is always suboptimal.
