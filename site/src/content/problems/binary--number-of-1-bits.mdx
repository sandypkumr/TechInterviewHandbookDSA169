## Problem Statement

Given a positive integer `n`, write a function that returns the number of set bits in its binary representation (also known as the Hamming weight).

 

**Example 1:**

**Input:** n = 11

**Output:** 3

**Explanation:**

The input binary string **1011** has a total of three set bits.

**Example 2:**

**Input:** n = 128

**Output:** 1

**Explanation:**

The input binary string **10000000** has a total of one set bit.

**Example 3:**

**Input:** n = 2147483645

**Output:** 30

**Explanation:**

The input binary string **1111111111111111111111111111101** has a total of thirty set bits.

 

**Constraints:**

- `1 <= n <= 2^31 - 1`

 

**Follow up:** If this function is called many times, how would you optimize it?

## Solution

```python
class Solution:
    def hammingWeight(self, n: int) -> int:
        count = 0
        while n:
            count += n & 1
            n >>= 1
        return count
```


## Explanation

### Approach: Check Least Significant Bit, Then Right Shift

At each step:
1. Read the rightmost bit using `n & 1`
2. Add it to the answer
3. Right shift `n` by one (`n >>= 1`) to process next bit

Repeat until `n` becomes 0.

### Why `n & 1` Works

- If last bit is `1`, then `n & 1 = 1`
- If last bit is `0`, then `n & 1 = 0`

So this gives exactly whether the current least significant bit is set.

### Step-by-Step Walkthrough

**Input:** `n = 11` (binary `1011`)

```
n = 1011, n&1 = 1 -> count = 1
n = 0101, n&1 = 1 -> count = 2
n = 0010, n&1 = 0 -> count = 2
n = 0001, n&1 = 1 -> count = 3
n = 0000 -> stop
```

Answer = `3`

### Code Breakdown

```python
count = 0
while n:
    count += n & 1
    n >>= 1
return count
```

This exactly scans every bit from right to left.

### Why This Complexity

- Time: O(number of bits in n) = O(log n)
- Space: O(1)

Note: A slightly faster variant is Brian Kernighan's trick (`n &= n-1`), which runs in O(number of set bits).
