## Problem Statement

Given the `root` of a binary search tree, and an integer `k`, return _the_ `k^th` _smallest value (**1-indexed**) of all the values of the nodes in the tree_.

 

**Example 1:**

```
Input: root = [3,1,4,null,2], k = 1
Output: 1
```

**Example 2:**

```
Input: root = [5,3,6,2,4,null,null,1], k = 3
Output: 3
```

 

**Constraints:**

- The number of nodes in the tree is `n`.
- `1 <= k <= n <= 10^4`
- `0 <= Node.val <= 10^4`

 

**Follow up:** If the BST is modified often (i.e., we can do insert and delete operations) and you need to find the kth smallest frequently, how would you optimize?

## Solution

```python
from typing import Optional


class TreeNode:
    def __init__(self, x):
        self.val = x
        self.left = None
        self.right = None


class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        stack = []
        while True:
            while root:
                stack.append(root)
                root = root.left
            root = stack.pop()
            k -= 1
            if k == 0:
                return root.val
            root = root.right
```


## Explanation

### Approach

This solution uses **Depth-First Search (DFS)** to explore all possible paths. It uses the **Two Pointers** technique to efficiently search through the data.

### Intuition

The solution breaks down the problem into manageable steps, processing elements systematically to build the final answer.

### Algorithm

1. **Main loop**: Continue until condition is met
2. **Move pointers**: Adjust left/right pointers based on current sum
3. **Return result**: Output the computed answer

### Why This Works

- **Two pointers optimization**: Eliminates need for nested loops by intelligently moving pointers
- **Correctness**: The algorithm handles all edge cases and produces the expected output
- **Efficiency**: Optimized for both time and space constraints
