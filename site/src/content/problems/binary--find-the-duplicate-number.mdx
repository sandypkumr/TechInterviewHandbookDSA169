## Problem Statement

Given an array of integers `nums` containing `n + 1` integers where each integer is in the range `[1, n]` inclusive.

There is only **one repeated number** in `nums`, return _this repeated number_.

You must solve the problem **without** modifying the array `nums` and using only constant extra space.

 

**Example 1:**

```
Input: nums = [1,3,4,2,2]
Output: 2
```

**Example 2:**

```
Input: nums = [3,1,3,4,2]
Output: 3
```

**Example 3:**

```
Input: nums = [3,3,3,3,3]
Output: 3
```

 

**Constraints:**

- `1 <= n <= 10^5`
- `nums.length == n + 1`
- `1 <= nums[i] <= n`
- All the integers in `nums` appear only **once** except for **precisely one integer** which appears **two or more** times.

 

**Follow up:**

- How can we prove that at least one duplicate number must exist in `nums`?
- Can you solve the problem in linear runtime complexity?

## Solution

```python
from typing import List


class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        slow = fast = nums[0]
        while True:
            slow = nums[slow]
            fast = nums[nums[fast]]
            if slow == fast:
                break
        slow = nums[0]
        while slow != fast:
            slow = nums[slow]
            fast = nums[fast]
        return slow
```


## Explanation

### Approach: Floyd's Cycle Detection

This solution treats the array like a linked list where each value points to the next index, then uses **Floyd's tortoise and hare** algorithm to find the cycle entry.

### Intuition

Because the numbers are in the range `1..n`, following `nums[i]` behaves like following a next pointer. A duplicate value means two different indices point into the same next position, which creates a cycle. The duplicate number is exactly the entrance to that cycle.

### Algorithm

Here's the step-by-step approach:

1. Start both `slow` and `fast` at `nums[0]`.
2. Move `slow` one step at a time and `fast` two steps at a time until they meet inside the cycle.
3. Reset `slow` to `nums[0]`, then move both pointers one step at a time until they meet again.
4. Return the meeting value, which is the duplicate number.

### Code Breakdown

The first loop finds an intersection point inside the cycle. The second loop uses the standard Floyd proof: starting one pointer from the beginning and one from the meeting point makes them meet at the cycle entrance

### Why This Works

- **Array-as-graph model:** Each index points to another index, creating a functional graph.
- **Duplicate creates a cycle:** Since there are `n + 1` positions but only `n` possible next values, some value must repeat, which forces a cycle.
- **Cycle entrance equals duplicate:** The node where the cycle begins is the repeated value reached by multiple indices.
- **Time-space optimization:** Balances time complexity gains with reasonable space usage


## Complexity

- **Time**: `O(n)`
- **Space**: `O(1)`
