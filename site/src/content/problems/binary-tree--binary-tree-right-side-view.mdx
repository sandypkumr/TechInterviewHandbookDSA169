## Problem Statement

Given the `root` of a binary tree, imagine yourself standing on the **right side** of it, return _the values of the nodes you can see ordered from top to bottom_.

 

**Example 1:**

**Input:** root = [1,2,3,null,5,null,4]

**Output:** [1,3,4]

**Explanation:**

**Example 2:**

**Input:** root = [1,2,3,4,null,null,null,5]

**Output:** [1,3,4,5]

**Explanation:**

**Example 3:**

**Input:** root = [1,null,3]

**Output:** [1,3]

**Example 4:**

**Input:** root = []

**Output:** []

 

**Constraints:**

- The number of nodes in the tree is in the range `[0, 100]`.
- `-100 <= Node.val <= 100`

## Solution

```python
from collections import deque
from typing import Optional, List


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        if not root:
            return []
        result = []
        queue = deque([root])
        while queue:
            level = []
            for _ in range(len(queue)):
                node = queue.popleft()
                level.append(node.val)
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
            result.append(level[-1])
        return result
```


## Explanation

### Approach: BFS

This solution uses **Breadth-First Search (BFS)** with a queue to explore states level by level.

### Intuition

The right side view contains exactly one value from each level: the node that appears last when that level is scanned from left to right. A level-order traversal naturally groups nodes by depth, so after processing one level we can simply record its final value.

### Algorithm

Here's the step-by-step approach:

1. Return an empty list if the tree is empty.
2. Start a queue with the root node.
3. For each level, pop all nodes currently in the queue, collect their values, and push their children for the next level.
4. Append the last value seen at that level to the result list.

### Code Breakdown

The queue always contains exactly the nodes of the current level before the inner loop starts. The temporary `level` list stores their values in left-to-right order, which makes `level[-1]` the visible node from the right

### Why This Works

- **Level separation:** BFS processes nodes depth by depth, so values from different levels never mix.
- **Rightmost selection:** Because children are enqueued left first, then right, the last value collected for a level is the rightmost node.
- **One answer per level:** Recording one value after each level produces the full right side view.

### Why This Works

- **Level-by-level exploration:** Guarantees we find the shortest path (if it's a path problem) and systematically covers all nodes
- **No nested loops:** Two pointers avoid O(n²) complexity by moving intelligently through the data
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
