## Problem Statement

Given an array of **distinct** integers `nums` and a target integer `target`, return _the number of possible combinations that add up to_ `target`.

The test cases are generated so that the answer can fit in a **32-bit** integer.

 

**Example 1:**

```
Input: nums = [1,2,3], target = 4
Output: 7
Explanation:
The possible combination ways are:
(1, 1, 1, 1)
(1, 1, 2)
(1, 2, 1)
(1, 3)
(2, 1, 1)
(2, 2)
(3, 1)
Note that different sequences are counted as different combinations.
```

**Example 2:**

```
Input: nums = [9], target = 3
Output: 0
```

 

**Constraints:**

- `1 <= nums.length <= 200`
- `1 <= nums[i] <= 1000`
- All the elements of `nums` are **unique**.
- `1 <= target <= 1000`

 

**Follow up:** What if negative numbers are allowed in the given array? How does it change the problem? What limitation we need to add to the question to allow negative numbers?

## Solution

```python
from typing import List


class Solution:
    def combinationSum4(self, nums: List[int], target: int) -> int:
        dp = [0] * (target + 1)
        dp[0] = 1
        for i in range(1, target + 1):
            for num in nums:
                if i - num >= 0:
                    dp[i] += dp[i - num]
        return dp[target]
```


## Explanation

### Approach: Dynamic Programming

This solution uses **Dynamic Programming** to build answers from smaller subproblems and avoid recomputation.

### Intuition

Let `dp[i]` represent the number of ordered combinations that add up to `i`. If we want to build a total of `i`, we can choose any number `num` as the last element, as long as `num <= i`. Every way to form `i - num` can then be extended with `num`, so we add `dp[i - num]` into `dp[i]`.

### Algorithm

Here's the step-by-step approach:

1. Create a DP array of length `target + 1` and set `dp[0] = 1` because there is exactly one way to make sum `0`: choose nothing.
2. Iterate `i` from `1` to `target`, treating `i` as the current total we want to build.
3. For each number in `nums`, if it can fit into the current total, add `dp[i - num]` to `dp[i]`.
4. After filling the table, return `dp[target]` as the number of ordered combinations.

### Code Breakdown

The outer loop fills the answer for every intermediate total from `1` up to `target`. The inner loop tries each candidate number as the last pick in the combination. Because `dp[i]` depends only on smaller totals, the table is built in a valid bottom-up order.

### Why This Works

- **Correct state definition:** `dp[i]` stores exactly the number of ordered ways to build total `i`.
- **Valid transition:** Every combination ending with `num` comes from a unique combination that builds `i - num`, so adding `dp[i - num]` counts all possibilities.
- **Bottom-up completeness:** By the time we compute `dp[i]`, all smaller states have already been computed, so no combination is missed.


## Complexity

- **Time**: `O(target × n)`
- **Space**: `O(target)`
