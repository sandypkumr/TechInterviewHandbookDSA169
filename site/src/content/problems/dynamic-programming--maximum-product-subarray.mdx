## Problem Statement

Given an integer array `nums`, find a subarray that has the largest product, and return _the product_.

The test cases are generated so that the answer will fit in a **32-bit** integer.

**Note** that the product of an array with a single element is the value of that element.

 

**Example 1:**

```
Input: nums = [2,3,-2,4]
Output: 6
Explanation: [2,3] has the largest product 6.
```

**Example 2:**

```
Input: nums = [-2,0,-1]
Output: 0
Explanation: The result cannot be 2, because [-2,-1] is not a subarray.
```

 

**Constraints:**

- `1 <= nums.length <= 2 * 10^4`
- `-10 <= nums[i] <= 10`
- The product of any subarray of `nums` is **guaranteed** to fit in a **32-bit** integer.

## Solution

```python
from typing import List


class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        if not nums:
            return 0
        max_product = min_product = result = nums[0]
        for i in range(1, len(nums)):
            if nums[i] < 0:
                max_product, min_product = min_product, max_product
            max_product = max(nums[i], max_product * nums[i])
            min_product = min(nums[i], min_product * nums[i])
            result = max(result, max_product)
        return result
```


## Explanation

### Approach: Dynamic Programming with Min-Max Tracking (Kadane Variant)

This solution uses a **modified Kadane's algorithm** that tracks both maximum and minimum products ending at each position. Unlike the standard Kadane's algorithm for sum, we need the minimum because a negative number can flip a large negative product into a large positive product.

### Intuition

**Key Insight:** The maximum product at position i could be:
1. The product ending at i-1 multiplied by nums[i]
2. nums[i] itself (restart)
3. The MINIMUM product at i-1 multiplied by nums[i] (negative × negative = positive!)

Unlike with sums, we must track both max and min because negatives matter.

### Algorithm

1. Track both `max_prod` and `min_prod` ending at current position
2. Calculate new candidates: `num`, `num × max_prod`, `num × min_prod`
3. Update max_prod and min_prod to the best candidates
4. Track the global maximum

### Why This Works

- **Handles Negatives:** By tracking minimums, we can leverage negative × negative = positive
- **O(n) Time:** Single pass through array
- **Kadane Variant:** Extends Kadane's algorithm for products instead of sums
