## Problem Statement

Given a string `s` representing a valid expression, implement a basic calculator to evaluate it, and return _the result of the evaluation_.

**Note:** You are **not** allowed to use any built-in function which evaluates strings as mathematical expressions, such as `eval()`.

 

**Example 1:**

```
Input: s = "1 + 1"
Output: 2
```

**Example 2:**

```
Input: s = " 2-1 + 2 "
Output: 3
```

**Example 3:**

```
Input: s = "(1+(4+5+2)-3)+(6+8)"
Output: 23
```

 

**Constraints:**

- `1 <= s.length <= 3 * 10^5`
- `s` consists of digits, `'+'`, `'-'`, `'('`, `')'`, and `' '`.
- `s` represents a valid expression.
- `'+'` is **not** used as a unary operation (i.e., `"+1"` and `"+(2 + 3)"` is invalid).
- `'-'` could be used as a unary operation (i.e., `"-1"` and `"-(2 + 3)"` is valid).
- There will be no two consecutive operators in the input.
- Every number and running calculation will fit in a signed 32-bit integer.

## Solution

```python
class Solution:
    def calculate(self, s: str) -> int:
        num_stack, sign_stack = [], []
        res, num, sign = 0, 0, '+'

        for i, c in enumerate(s):
            if c.isdigit():
                num = num * 10 + int(c)
            if c in '+-' or i == len(s) - 1:
                res = res + num if sign == '+' else res - num
                num, sign = 0, c
            if c == '(':
                num_stack.append(res)
                sign_stack.append(sign)
                res, num, sign = 0, 0, '+'
            if c == ')':
                res = res + num if sign == '+' else res - num
                prv_res = num_stack.pop()
                res = prv_res + res if sign_stack.pop() == '+' else prv_res - res
                num = 0
        return res
```


## Explanation

### Approach: Stack-based Expression Evaluation

This solution uses a **stack** to evaluate the expression while preserving operator precedence and parentheses.

### Intuition

The expression only contains `+`, `-`, and parentheses, so we can keep a running total for the current level of parentheses. When we enter a new parenthesized expression, we save the previous result and sign on stacks. When we close the parentheses, we evaluate the inner result and merge it back into the outer context.

### Algorithm

Here's the step-by-step approach:

1. Scan the string character by character, building multi-digit numbers as needed.
2. When a `+` or `-` is reached, apply the previous sign to the current number and add it into `res`.
3. When `(` appears, push the current `res` and `sign`, then reset the inner-expression state. When `)` appears, finish the inner expression and combine it with the saved outer state.
4. Return the final accumulated result after the scan finishes.

### Code Breakdown

`num_stack` stores the accumulated result before each opening parenthesis, and `sign_stack` stores whether that subexpression should be added or subtracted when it closes. This lets the parser evaluate nested expressions in one pass without building an explicit syntax tree.

### Why This Works

- **Correct local evaluation:** At any point, `res` stores the value of the current parenthesis level processed so far.
- **Proper nesting support:** Saving outer state before `(` and restoring it after `)` correctly models recursive expression structure.
- **Single-pass parsing:** Every character is processed once, and each parenthesis pair is resolved when closed.


## Complexity

- **Time**: `O(n)`
- **Space**: `O(n)`
