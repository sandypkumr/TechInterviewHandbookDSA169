## Problem Statement

Given an integer array `nums` and an integer `k`, return _the_ `k^th` _largest element in the array_.

Note that it is the `k^th` largest element in the sorted order, not the `k^th` distinct element.

Can you solve it without sorting?

 

**Example 1:**

```
Input: nums = [3,2,1,5,6,4], k = 2
Output: 5
```
**Example 2:**

```
Input: nums = [3,2,3,1,2,4,5,5,6], k = 4
Output: 4
```

 

**Constraints:**

- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        min_val, max_val = min(nums), max(nums)
        count = [0] * (max_val - min_val + 1)
        for num in nums:
            count[num - min_val] += 1
        for i in range(max_val, min_val - 1, -1):
            k -= count[i - min_val]
            if k <= 0:
                return i
```


## Explanation

### Approach: Counting Array (Bucket by Value Range)

This implementation does **not** use a heap. It uses frequency counting across the value range.

Steps:
1. Find `min_val` and `max_val`
2. Build `count` array where `count[x - min_val]` is frequency of value `x`
3. Iterate values from `max_val` downward, subtracting frequencies from `k`
4. The first value where `k <= 0` is the k-th largest

### Step-by-Step Example

Input: `nums=[3,2,1,5,6,4]`, `k=2`

- Value range: 1..6
- Frequencies: `1:1, 2:1, 3:1, 4:1, 5:1, 6:1`

Walk from high to low:
- at `6`: `k = 2 - 1 = 1`
- at `5`: `k = 1 - 1 = 0` -> return `5`

Answer is `5`.

### Code Breakdown

```python

```

## Complexity

- **Time**: `O(n + R)` where `R = max(nums) - min(nums) + 1`
- **Space**: `O(R)`
