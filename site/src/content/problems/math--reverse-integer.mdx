## Problem Statement

Given a signed 32-bit integer `x`, return `x`_ with its digits reversed_. If reversing `x` causes the value to go outside the signed 32-bit integer range `[-2^31, 2^31 - 1]`, then return `0`.

**Assume the environment does not allow you to store 64-bit integers (signed or unsigned).**

 

**Example 1:**

```
Input: x = 123
Output: 321
```

**Example 2:**

```
Input: x = -123
Output: -321
```

**Example 3:**

```
Input: x = 120
Output: 21
```

 

**Constraints:**

- `-2^31 <= x <= 2^31 - 1`

## Solution

```python
class Solution:
    def reverse(self, x: int) -> int:
        if x == 0:
            return 0
        sign = 1 if x > 0 else -1
        x = abs(x)
        res = 0
        while x:
            res = res * 10 + x % 10
            x //= 10
        res *= sign
        if res < -2 ** 31 or res > 2 ** 31 - 1:
            return 0
        return res
```


## Explanation

### Approach: Digit Reversal

This solution reverses digits one by one and rebuilds the result while preserving sign constraints.

### Intuition

Peel the digits off from right to left and append them to a new number from left to right. After reversing, restore the original sign and reject the result if it falls outside the 32-bit signed integer range.

### Algorithm

Here's the step-by-step approach:

1. Return `0` immediately if the input is `0`.
2. Save the sign, then work with the absolute value of `x`.
3. Repeatedly extract the last digit with `% 10` and append it to `res` using `res = res * 10 + digit`.
4. Reapply the sign and return `0` if the reversed value is outside the 32-bit signed range; otherwise return it.

### Code Breakdown

The loop consumes one digit per iteration by using integer division to drop the last digit. The overflow check is performed after rebuilding the reversed value, matching the problem's requirement to return `0` when the reversed integer does not fit.

### Why This Works

- **Correct digit order:** Taking digits from the end of `x` and appending them to `res` reconstructs the number in reverse.
- **Sign preservation:** Separating and restoring the sign keeps negative and positive inputs consistent.
- **Overflow safety:** The final bounds check ensures only valid 32-bit signed results are returned.
