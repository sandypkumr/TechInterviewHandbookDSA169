## Problem Statement

Given an unsorted integer array `nums`. Return the _smallest positive integer_ that is _not present_ in `nums`.

You must implement an algorithm that runs in `O(n)` time and uses `O(1)` auxiliary space.

 

**Example 1:**

```
Input: nums = [1,2,0]
Output: 3
Explanation: The numbers in the range [1,2] are all in the array.
```

**Example 2:**

```
Input: nums = [3,4,-1,1]
Output: 2
Explanation: 1 is in the array but 2 is missing.
```

**Example 3:**

```
Input: nums = [7,8,9,11,12]
Output: 1
Explanation: The smallest positive integer 1 is missing.
```

 

**Constraints:**

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`

## Solution

```python
from typing import List


class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        nums = set(nums)
        for i in range(1, len(nums) + 2):
            if i not in nums:
                return i
```


## Explanation

### Approach: Hash-based approach

This implementation converts the array into a set so membership checks become O(1) on average. Then it scans upward from `1` until it finds the first positive integer that is missing.

### Intuition

The first missing positive must lie in the range `1` through `len(nums) + 1`. After putting every number into a set, we can test each candidate in increasing order and return the first value that is absent.

### Algorithm

1. Convert `nums` into a set
2. Check integers from `1` to `len(nums) + 1`
3. Return the first value that is not in the set
4. That value is the smallest missing positive

### Code Breakdown

- `nums = set(nums)`: enables fast membership checks
- `for i in range(1, len(nums) + 2)`: covers every possible answer
- `if i not in nums`: the first missing positive is found

### Why This Works

- All present positive values are recorded in the set
- Scanning upward from `1` guarantees the first missing value is the smallest missing positive
- The pigeonhole argument ensures the answer is never larger than `len(nums) + 1`
- This implementation is simple and correct, though it uses extra space compared with the optimal in-place solution


## Complexity

- **Time**: `O(n)`
- **Space**: `O(n)`
