## Problem Statement

Given an integer array `nums`, return `true` if any value appears **at least twice** in the array, and return `false` if every element is distinct.

 

**Example 1:**

**Input:** nums = [1,2,3,1]

**Output:** true

**Explanation:**

The element 1 occurs at the indices 0 and 3.

**Example 2:**

**Input:** nums = [1,2,3,4]

**Output:** false

**Explanation:**

All elements are distinct.

**Example 3:**

**Input:** nums = [1,1,1,3,3,4,3,2,4,2]

**Output:** true

 

**Constraints:**

- `1 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

## Solution

```python
from typing import List


class Solution:
    def containsDuplicate(self, nums: List[int]) -> bool:
        # nums_set = set()
        # for num in nums:
        #     if num in nums_set:
        #         return True
        #     nums_set.add(num)
        # return False
        return len(nums) != len(set(nums))
```


## Explanation

### Approach: Set for Duplicate Detection

The simplest and most elegant solution: **convert the array to a set and compare sizes**. If any duplicates exist, the set will be smaller than the original array.

### Why This Works

A set only stores unique values. If `len(set(nums)) < len(nums)`, then some values appeared more than once.

### The Intuition

Sets automatically discard duplicates. So:
- **Original array has duplicates:** `len(nums) != len(set(nums))`  ✓ Return True
- **No duplicates:** `len(nums) == len(set(nums))` → Return False

### Single Line Solution

```python
return len(nums) != len(set(nums))
```

That's it! One line, O(n) time, O(n) space.

### Alternative: Hash Set with Early Exit

If you want to find the first duplicate early without creating the full set:

```python
seen = set()
for num in nums:
    if num in seen:
        return True  # Found a duplicate!
    seen.add(num)
return False
```

**Why this matters:** If the array has duplicates near the start, we exit immediately instead of processing the entire array.

### Why O(n) is Optimal

You need to look at every element at least once, so O(n) is the theoretical minimum. The set approach achieves this minimum while being incredibly concise!
