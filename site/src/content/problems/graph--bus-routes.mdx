## Problem Statement

You are given an array `routes` representing bus routes where `routes[i]` is a bus route that the `i^th` bus repeats forever.

- For example, if `routes[0] = [1, 5, 7]`, this means that the `0^th` bus travels in the sequence `1 -> 5 -> 7 -> 1 -> 5 -> 7 -> 1 -> ...` forever.

You will start at the bus stop `source` (You are not on any bus initially), and you want to go to the bus stop `target`. You can travel between bus stops by buses only.

Return _the least number of buses you must take to travel from _`source`_ to _`target`. Return `-1` if it is not possible.

 

**Example 1:**

```
Input: routes = [[1,2,7],[3,6,7]], source = 1, target = 6
Output: 2
Explanation: The best strategy is take the first bus to the bus stop 7, then take the second bus to the bus stop 6.
```

**Example 2:**

```
Input: routes = [[7,12],[4,5,15],[6],[15,19],[9,12,13]], source = 15, target = 12
Output: -1
```

 

 

**Constraints:**

- `1 <= routes.length <= 500`.
- `1 <= routes[i].length <= 10^5`
- All the values of `routes[i]` are **unique**.
- `sum(routes[i].length) <= 10^5`
- `0 <= routes[i][j] < 10^6`
- `0 <= source, target < 10^6`

## Solution

```python
from collections import defaultdict, deque
from typing import List


class Solution:
    def numBusesToDestination(self, routes: List[List[int]], source: int, target: int) -> int:
        graph = defaultdict(set)
        for i, route in enumerate(routes):
            for stop in route:
                graph[stop].add(i)

        queue = deque([(source, 0)])
        visited = {source}
        while queue:
            stop, count = queue.popleft()
            if stop == target:
                return count
            for bus in graph[stop]:
                for next_stop in routes[bus]:
                    if next_stop not in visited:
                        visited.add(next_stop)
                        queue.append((next_stop, count + 1))
                routes[bus] = []
        return -1
```


## Explanation

### Approach: BFS

This solution uses **Breadth-First Search (BFS)** with a queue to explore states level by level.

### Intuition

Each bus ride increases the answer by exactly one, so BFS is a natural fit. From a stop, we can board any bus serving that stop and then reach every other stop on that route in one more bus ride.

### Algorithm

Here's the step-by-step approach:

1. Build a map from each stop to the set of bus routes that include it.
2. Start BFS from the `source` stop with bus count `0`.
3. For each stop, explore every bus route available there, then enqueue all stops on those routes with count `+1` if they have not been visited.
4. Clear a route once it has been used so the same bus is not processed repeatedly, and return the count when `target` is reached.

### Code Breakdown

`graph` connects stops to route indices, which avoids scanning every route from scratch. The `visited` set prevents revisiting stops, and `routes[bus] = []` acts like marking a bus route as consumed so it is expanded only once

### Why This Works

- **BFS by bus count:** Each layer of the search corresponds to taking one more bus.
- **Complete transfers:** From any stop, exploring all buses serving that stop covers every valid next transfer.
- **No redundant route expansion:** Clearing a route after using it keeps the traversal efficient without losing any reachable stops.

### Why This Works

- **Level-by-level exploration:** Guarantees we find the shortest path (if it's a path problem) and systematically covers all nodes
- **O(1) lookups:** Hash maps provide constant-time insertion and retrieval, critical for efficient searching
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
