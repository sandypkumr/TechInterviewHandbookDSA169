## Problem Statement

Given a **sorted** integer array `arr`, two integers `k` and `x`, return the `k` closest integers to `x` in the array. The result should also be sorted in ascending order.

An integer `a` is closer to `x` than an integer `b` if:

- `|a - x| < |b - x|`, or
- `|a - x| == |b - x|` and `a < b`

 

**Example 1:**

**Input:** arr = [1,2,3,4,5], k = 4, x = 3

**Output:** [1,2,3,4]

**Example 2:**

**Input:** arr = [1,1,2,3,4,5], k = 4, x = -1

**Output:** [1,1,2,3]

 

**Constraints:**

- `1 <= k <= arr.length`
- `1 <= arr.length <= 10^4`
- `arr` is sorted in **ascending** order.
- `-10^4 <= arr[i], x <= 10^4`

## Solution

```python
from typing import List


class Solution:
    def findClosestElements(self, arr: List[int], k: int, x: int) -> List[int]:
        if k == len(arr):
            return arr
        if x <= arr[0]:
            return arr[:k]
        if x >= arr[-1]:
            return arr[-k:]
        left, right = 0, len(arr) - k
        while left < right:
            mid = left + (right - left) // 2
            if x - arr[mid] > arr[mid + k] - x:
                left = mid + 1
            else:
                right = mid
        return arr[left:left + k]
```


## Explanation

### Approach: Binary Search

This solution uses **Binary Search** to efficiently find the target by halving the search space with each iteration.

### Intuition

The answer must be a contiguous window of length `k` in the sorted array. Instead of choosing elements one by one, we binary search the starting index of that window by comparing which side is farther from `x`: `arr[mid]` or `arr[mid + k]`.

### Algorithm

Here's the step-by-step approach:

1. Handle the easy boundary cases where `x` lies entirely to the left or right of the array, or where `k` already equals the array length.
2. Binary search the window start in the range `[0, len(arr) - k]`.
3. At each `mid`, compare `x - arr[mid]` with `arr[mid + k] - x` to decide whether the better window starts to the right or left.
4. Return the length-`k` slice starting at the final `left` index.

### Code Breakdown

The search does not look for `x` directly. It looks for the best window start, shrinking the candidate range until only one starting position remains.

### Why This Works

- **Contiguous-answer property:** In a sorted array, the `k` closest elements always form a consecutive block.
- **Window comparison:** If `arr[mid]` is farther from `x` than `arr[mid + k]`, the optimal window must start to the right of `mid`.
- **Binary search validity:** That comparison is monotonic across start positions, so binary search can find the best window efficiently.
- **No nested loops:** Two pointers avoid O(n²) complexity by moving intelligently through the data
- **Handles edge cases:** Correctly processes boundary conditions and special inputs
- **Time-space optimization:** Balances time complexity gains with reasonable space usage
