## Problem Statement

You are given a 2D grid where:

Each cell in the grid can be one of the following:
'*': Your starting position. There is exactly one '*' in the grid.
'#': A food cell. There may be multiple food cells.
'O': An open cell you can walk through.
'X': A blocked cell you cannot walk through.
You need to find the shortest path from your starting position ('*') to any food cell ('#') in the grid. You can move
 up, down, left, or right. Return the number of steps in the shortest path, or -1 if no path exists.


## Examples

Example 1:

Example 1:
Input: grid =   [["X","X","X","X","X","X"],
                 ["X","*","O","O","O","X"],
                 ["X","O","O","#","O","X"],
                 ["X","X","X","X","X","X"]]
Output: 3
Explanation: It takes 3 steps to reach the food.


## Solution

```python
from typing import List


class Solution:
    def getFood(self, grid: List[List[str]]) -> int:
        if not grid:
            return -1
        rows, cols = len(grid), len(grid[0])
        queue = [(i, j) for i in range(rows) for j in range(cols) if grid[i][j] == '*']
        steps = 0
        while queue:
            steps += 1
            for _ in range(len(queue)):
                i, j = queue.pop(0)
                for x, y in [(i + 1, j), (i - 1, j), (i, j + 1), (i, j - 1)]:
                    if 0 <= x < rows and 0 <= y < cols and grid[x][y] != 'X':
                        if grid[x][y] == '#':
                            return steps
                        grid[x][y] = 'X'
                        queue.append((x, y))
        return -1
```


## Explanation

### Approach: BFS

This solution uses **Breadth-First Search** from the starting cell because every move has equal cost and we want the shortest path.

### Intuition

In a grid where you can move one step at a time, BFS finds the nearest target by exploring all cells at distance `d` before any cell at distance `d + 1`. That means the first time we see `'#'`, we have found the shortest route to food.

### Algorithm

Here's the step-by-step approach:

1. Locate the starting cell `'*'` and place it in the queue.
2. Expand the search level by level, exploring the four neighboring cells of each position.
3. Skip walls and already visited cells; when a neighbor is food `'#'`, return the current step count.
4. If the queue empties without reaching food, return `-1`.

### Code Breakdown

The code marks visited cells in-place by turning them into `'X'`, which avoids a separate visited set. The `steps` counter increases once per BFS layer, so it stays aligned with path length.

### Why This Works

- **Shortest-path guarantee:** BFS explores positions in increasing order of distance from the start.
- **No revisits:** Marking visited cells prevents cycles and redundant work.
- **First food found is optimal:** The first reachable food cell encountered must be the minimum-step answer.
