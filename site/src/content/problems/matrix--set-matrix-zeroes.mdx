## Problem Statement

Given an `m x n` integer matrix `matrix`, if an element is `0`, set its entire row and column to `0`'s.

You must do it in place.

 

**Example 1:**

```
Input: matrix = [[1,1,1],[1,0,1],[1,1,1]]
Output: [[1,0,1],[0,0,0],[1,0,1]]
```

**Example 2:**

```
Input: matrix = [[0,1,2,0],[3,4,5,2],[1,3,1,5]]
Output: [[0,0,0,0],[0,4,5,0],[0,3,1,0]]
```

 

**Constraints:**

- `m == matrix.length`
- `n == matrix[0].length`
- `1 <= m, n <= 200`
- `-2^31 <= matrix[i][j] <= 2^31 - 1`

 

**Follow up:**

- A straightforward solution using `O(mn)` space is probably a bad idea.
- A simple improvement uses `O(m + n)` space, but still not the best solution.
- Could you devise a constant space solution?

## Solution

```python
from typing import List


class Solution:
    def setZeroes(self, matrix: List[List[int]]) -> None:
        """
        Do not return anything, modify matrix in-place instead.
        """
        rows, cols = set(), set()
        for i in range(len(matrix)):
            for j in range(len(matrix[0])):
                if matrix[i][j] == 0:
                    rows.add(i)
                    cols.add(j)
        for i in range(len(matrix)):
            for j in range(len(matrix[0])):
                if i in rows or j in cols:
                    matrix[i][j] = 0
```


## Explanation

### Approach: Row and Column Marking

This solution records which rows and columns must become zero, then performs a second pass to update the matrix in place.

### Intuition

If a cell contains `0`, then its entire row and entire column must be zeroed. Instead of modifying the matrix immediately and accidentally spreading extra zeros, we first remember the affected row indices and column indices, then apply the changes afterward.

### Algorithm

Here's the step-by-step approach:

1. Traverse the matrix and store every row index and column index that contains a zero.
2. Traverse the matrix again.
3. If a cell belongs to a marked row or a marked column, set it to zero.
4. Finish in place without returning anything.

### Code Breakdown

The `rows` and `cols` sets separate discovery from modification. That avoids corrupting the scan by turning additional cells to zero before the algorithm has finished identifying the original zero positions

### Why This Works

- **Correct dependency tracking:** Every original zero contributes exactly one row marker and one column marker.
- **No accidental propagation:** Delaying writes until the second pass prevents newly written zeros from triggering extra row/column clears.
- **Complete coverage:** Any cell sharing a row or column with an original zero is guaranteed to be reset.


## Complexity

- **Time**: `O(m×n)`
- **Space**: `O(1)`
