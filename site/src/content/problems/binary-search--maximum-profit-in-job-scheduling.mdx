## Problem Statement

We have `n` jobs, where every job is scheduled to be done from `startTime[i]` to `endTime[i]`, obtaining a profit of `profit[i]`.

You're given the `startTime`, `endTime` and `profit` arrays, return the maximum profit you can take such that there are no two jobs in the subset with overlapping time range.

If you choose a job that ends at time `X` you will be able to start another job that starts at time `X`.

 

**Example 1:**

****

```
Input: startTime = [1,2,3,3], endTime = [3,4,5,6], profit = [50,10,40,70]
Output: 120
Explanation: The subset chosen is the first and fourth job. 
Time range [1-3]+[3-6] , we get profit of 120 = 50 + 70.
```

**Example 2:**

** **

```
Input: startTime = [1,2,3,4,6], endTime = [3,5,10,6,9], profit = [20,20,100,70,60]
Output: 150
Explanation: The subset chosen is the first, fourth and fifth job. 
Profit obtained 150 = 20 + 70 + 60.
```

**Example 3:**

****

```
Input: startTime = [1,1,1], endTime = [2,3,4], profit = [5,6,4]
Output: 6
```

 

**Constraints:**

- `1 <= startTime.length == endTime.length == profit.length <= 5 * 10^4`
- `1 <= startTime[i] < endTime[i] <= 10^9`
- `1 <= profit[i] <= 10^4`

## Solution

```python
from bisect import bisect
from typing import List


class Solution:
    def jobScheduling(self, startTime: List[int], endTime: List[int], profit: List[int]) -> int:
        jobs = sorted(zip(startTime, endTime, profit), key=lambda x: x[1])
        dp = [(0, 0)]
        for s, e, p in jobs:
            i = bisect(dp, (s + 1,)) - 1
            if dp[i][1] + p > dp[-1][1]:
                dp.append((e, dp[i][1] + p))
        return dp[-1][1]
```


## Explanation

### Approach: Dynamic Programming + Binary Search

This solution sorts jobs by end time, then combines **Dynamic Programming** with **binary search** to quickly find the best compatible previous job.

### Intuition

If we process jobs in order of ending time, then for each job we only need to know the best profit achievable before its start time. The `dp` list stores pairs of `(end_time, best_profit_so_far)`, and binary search lets us jump directly to the last non-overlapping job instead of scanning backward.

### Algorithm

Here's the step-by-step approach:

1. Zip and sort the jobs by their end time.
2. Maintain `dp`, where each entry stores an end time and the maximum profit achievable up to that time.
3. For each job `(s, e, p)`, binary search `dp` to find the latest entry whose end time is not greater than `s`, then compute the profit of taking this job.
4. If that profit improves on the current best, append `(e, new_profit)` to `dp`, and return the best profit from the last entry.

### Code Breakdown

The sorted jobs guarantee that every compatible earlier profit is already represented in `dp`. The `bisect` call finds the best previously finished schedule in `O(log n)`, and the append happens only when the current job improves the global best.

### Why This Works

- **Optimal substructure:** The best schedule ending with a job equals that job's profit plus the best compatible profit before its start.
- **Binary-search lookup:** Because `dp` is ordered by end time, the last compatible state can be found efficiently.
- **Monotonic DP states:** Each appended entry improves the best profit so far, so `dp[-1][1]` is always the global optimum seen up to that point.
- **Correctness:** The algorithm handles all edge cases and produces the expected output
- **Optimization:** Balances time complexity gains with reasonable space usage
