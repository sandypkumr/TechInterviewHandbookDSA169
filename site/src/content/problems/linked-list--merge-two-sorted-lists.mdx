## Problem Statement

You are given the heads of two sorted linked lists `list1` and `list2`.

Merge the two lists into one **sorted** list. The list should be made by splicing together the nodes of the first two lists.

Return _the head of the merged linked list_.

 

**Example 1:**

```
Input: list1 = [1,2,4], list2 = [1,3,4]
Output: [1,1,2,3,4,4]
```

**Example 2:**

```
Input: list1 = [], list2 = []
Output: []
```

**Example 3:**

```
Input: list1 = [], list2 = [0]
Output: [0]
```

 

**Constraints:**

- The number of nodes in both lists is in the range `[0, 50]`.
- `-100 <= Node.val <= 100`
- Both `list1` and `list2` are sorted in **non-decreasing** order.

## Solution

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode()
        current = dummy
        while list1 and list2:
            if list1.val < list2.val:
                current.next = list1
                list1 = list1.next
            else:
                current.next = list2
                list2 = list2.next
            current = current.next
        current.next = list1 or list2
        return dummy.next
```


## Explanation

### Approach: Iterative Merge with Two Pointers

Both lists are already sorted, so we merge exactly like merge step in merge sort.
Keep pointers on `list1` and `list2`, always attach the smaller node to the result, and move that list forward.

### Why Dummy Node Helps

Using a dummy head avoids special-casing the first insertion.
`current` always points to the last node in merged list.

### Step-by-Step Example

Input:
- `list1 = 1 -> 2 -> 4`
- `list2 = 1 -> 3 -> 4`

Process:
- compare `1` and `1` -> take from `list2`
- compare `1` and `3` -> take from `list1`
- compare `2` and `3` -> take from `list1`
- compare `4` and `3` -> take from `list2`
- compare `4` and `4` -> take from `list2`
- `list2` exhausted -> append remaining `list1` (`4`)

Result: `1 -> 1 -> 2 -> 3 -> 4 -> 4`

### Code Breakdown

```python
dummy = ListNode()
current = dummy
while list1 and list2:
    if list1.val < list2.val:
        current.next = list1
        list1 = list1.next
    else:

    current = current.next
current.next = list1 or list2
```

`current.next = list1 or list2` attaches whichever list still has nodes.

### Why This Works

- At each step we pick the smallest available head, preserving sorted order
- Every node is visited once
- No new nodes are created; we only relink pointers
