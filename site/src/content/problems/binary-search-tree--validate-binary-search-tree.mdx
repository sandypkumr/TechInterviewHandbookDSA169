## Problem Statement

Given the `root` of a binary tree, _determine if it is a valid binary search tree (BST)_.

A **valid BST** is defined as follows:

- The left subtree of a node contains only nodes with keys **strictly less than** the node's key.
- The right subtree of a node contains only nodes with keys **strictly greater than** the node's key.
- Both the left and right subtrees must also be binary search trees.

 

**Example 1:**

```
Input: root = [2,1,3]
Output: true
```

**Example 2:**

```
Input: root = [5,1,4,null,null,3,6]
Output: false
Explanation: The root node's value is 5 but its right child's value is 4.
```

 

**Constraints:**

- The number of nodes in the tree is in the range `[1, 10^4]`.
- `-2^31 <= Node.val <= 2^31 - 1`

## Solution

```python
from typing import Optional


class TreeNode:
    def __init__(self, x):
        self.val = x
        self.left = None
        self.right = None


class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        def helper(node, lower=float('-inf'), upper=float('inf')):
            if not node:
                return True

            val = node.val
            if val <= lower or val >= upper:
                return False

            return helper(node.right, val, upper) and helper(node.left, lower, val)

        return helper(root)
```


## Explanation

### Approach: DFS with Valid Value Range Propagation

For each node, keep allowed value bounds:
- all nodes in left subtree must be `< node.val`
- all nodes in right subtree must be `> node.val`

So each recursive call receives `(lower, upper)` constraints and checks whether current value is inside them.

### Code Logic

```python
if val <= lower or val >= upper:
    return False
return helper(node.right, val, upper) and helper(node.left, lower, val)
```

At root, bounds are `(-inf, +inf)`.

### Why Simple Parent-Only Check Fails

A node can violate BST rules relative to an ancestor, not just its direct parent.
Range propagation correctly captures all ancestor constraints.

### Why This Works

- Every node is validated against full allowable interval
- Bounds tighten as recursion goes deeper
- One pass over nodes gives O(n) time and O(h) recursion stack
