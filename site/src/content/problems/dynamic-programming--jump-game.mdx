## Problem Statement

You are given an integer array `nums`. You are initially positioned at the array's **first index**, and each element in the array represents your maximum jump length at that position.

Return `true`_ if you can reach the last index, or _`false`_ otherwise_.

 

**Example 1:**

```
Input: nums = [2,3,1,1,4]
Output: true
Explanation: Jump 1 step from index 0 to 1, then 3 steps to the last index.
```

**Example 2:**

```
Input: nums = [3,2,1,0,4]
Output: false
Explanation: You will always arrive at index 3 no matter what. Its maximum jump length is 0, which makes it impossible to reach the last index.
```

 

**Constraints:**

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 10^5`

## Solution

```python
from typing import List


class Solution:
    def canJump(self, nums: List[int]) -> bool:
        can_start_here = len(nums) - 1
        for i in range(len(nums) - 2, -1, -1):
            if i + nums[i] >= can_start_here:
                can_start_here = i
        return can_start_here == 0
```


## Explanation

### Approach: Greedy Algorithm

This solution uses a **Greedy** strategy to make locally optimal decisions that lead to the global optimum.

### Intuition

Instead of asking, "How far can I go from the start?", this solution works backward and asks, "What is the leftmost position that can still reach the end?" If an index `i` can jump to that position or beyond, then `i` becomes the new leftmost good position. In the end, we only need to check whether index `0` is good.

### Algorithm

Here's the step-by-step approach:

1. Start with `can_start_here` set to the last index, since the last index can trivially reach itself.
2. Scan the array from right to left.
3. Whenever `i + nums[i]` reaches `can_start_here` or goes beyond it, update `can_start_here = i`.
4. After the scan, return whether `can_start_here` moved all the way back to index `0`.

### Code Breakdown

The loop checks each position exactly once. The variable `can_start_here` tracks the nearest index to the left that must be reachable. Each successful update shrinks the problem, because now we only need to reach an even earlier index.

### Why This Works

- **Greedy invariant:** `can_start_here` is always the leftmost index currently known to reach the end.
- **Backward reduction:** If index `i` can reach `can_start_here`, then `i` can also reach the end, so it is safe to replace the target with `i`.
- **Final check:** If the target is reduced to `0`, then the start can reach the end; otherwise it cannot.


## Complexity

- **Time**: `O(n)`
- **Space**: `O(1)`
