## Problem Statement

Given a **non-empty** array of integers `nums`, every element appears _twice_ except for one. Find that single one.

You must implement a solution with a linear runtime complexity and use only constant extra space.

 

**Example 1:**

**Input:** nums = [2,2,1]

**Output:** 1

**Example 2:**

**Input:** nums = [4,1,2,1,2]

**Output:** 4

**Example 3:**

**Input:** nums = [1]

**Output:** 1

 

**Constraints:**

- `1 <= nums.length <= 3 * 10^4`
- `-3 * 10^4 <= nums[i] <= 3 * 10^4`
- Each element in the array appears twice except for one element which appears only once.

## Solution

```python
from functools import reduce
from typing import List


class Solution:
    def singleNumber(self, nums: List[int]) -> int:
        return reduce(lambda x, y: x ^ y, nums)
```


## Explanation

### Approach: XOR Cancellation

Use XOR over all elements. The duplicate numbers cancel each other, leaving only the single number.

### XOR Properties Used

- `a ^ a = 0` (same numbers cancel)
- `a ^ 0 = a` (identity)
- XOR is commutative and associative, so order does not matter

### Step-by-Step Walkthrough

**Input:** `[4,1,2,1,2]`

```
acc = 4
acc = 4 ^ 1 = 5
acc = 5 ^ 2 = 7
acc = 7 ^ 1 = 6   (the previous 1 got canceled)
acc = 6 ^ 2 = 4   (the previous 2 got canceled)
```

Answer = `4`

Conceptually this is:

`4 ^ 1 ^ 2 ^ 1 ^ 2 = 4 ^ (1^1) ^ (2^2) = 4 ^ 0 ^ 0 = 4`

### Code Breakdown

```python
return reduce(lambda x, y: x ^ y, nums)
```

`reduce` applies XOR cumulatively across the array.

Equivalent expanded version:

```python
ans = 0
for num in nums:
    ans ^= num
return ans
```

### Why This is Optimal

- O(n) time: one pass over the array
- O(1) extra space: only one accumulator
- Exactly matches the constraint requirements


## Complexity

- **Time**: `O(n)`
- **Space**: `O(1)`
