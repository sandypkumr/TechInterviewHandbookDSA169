## Problem Statement

Given the `root` of a binary tree, the value of a target node `target`, and an integer `k`, return _an array of the values of all nodes that have a distance _`k`_ from the target node._

You can return the answer in **any order**.

 

**Example 1:**

```
Input: root = [3,5,1,6,2,0,8,null,null,7,4], target = 5, k = 2
Output: [7,4,1]
Explanation: The nodes that are a distance 2 from the target node (with value 5) have values 7, 4, and 1.
```

**Example 2:**

```
Input: root = [1], target = 1, k = 3
Output: []
```

 

**Constraints:**

- The number of nodes in the tree is in the range `[1, 500]`.
- `0 <= Node.val <= 500`
- All the values `Node.val` are **unique**.
- `target` is the value of one of the nodes in the tree.
- `0 <= k <= 1000`

## Solution

```python
from typing import List


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def distanceK(self, root: TreeNode, target: TreeNode, k: int) -> List[int]:
        def dfs(node, parent):
            if not node:
                return
            node.parent = parent
            dfs(node.left, node)
            dfs(node.right, node)

        dfs(root, None)
        cur_arr = [target]
        visited = {target}
        for _ in range(k):
            new_arr = []
            for node in cur_arr:
                for neighbour in node.left, node.right, node.parent:
                    if neighbour and neighbour not in visited:
                        visited.add(neighbour)
                        new_arr.append(neighbour)
            cur_arr = new_arr
        return [x.val for x in cur_arr]
```


## Explanation

### Approach: Build Parent Links, Then BFS from Target

Tree edges are naturally child-directional, but distance-k search needs movement in all directions (left, right, parent).

So the solution does two phases:

1. DFS to attach `parent` reference to each node
2. BFS-like expansion from `target` for exactly `k` layers using neighbors `(left, right, parent)`

### Phase 1: Parent Mapping

```python
def dfs(node, parent):
    if not node:
        return
    node.parent = parent
    dfs(node.left, node)
    dfs(node.right, node)
```

### Phase 2: k-Level Expansion

`cur_arr` holds current frontier. Repeat `k` times:
- gather unvisited neighbors into `new_arr`
- set `cur_arr = new_arr`

After k rounds, `cur_arr` contains nodes at distance k.

### Why `visited` is Needed

Once parent links exist, graph becomes undirected-like; without `visited`, traversal would bounce back and loop.

### Why This Works

- Parent augmentation enables full graph-neighbor traversal
- Layered expansion guarantees exact distance by BFS levels
- Each node visited at most once -> O(n) time, O(n) space


## Complexity

- **Time**: `O(n)`
- **Space**: `O(n)`
